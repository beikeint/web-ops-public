# 数据分析与选题技能

> 触发指令：`[客户名] 开始本月运营`  
> 执行周期：每月 Day 1-5  
> 人工确认：⏸️ 选题需确认后才进入内容执行

---

## MCP 调用链

```
Step 1: 加载客户
        → client-manager.search_clients(客户名)
        → client-manager.get_client(client_id)
        → 获取：域名、语言列表、竞品列表、关键词矩阵、服务方案

Step 2: 并行数据采集
        ┌→ site-monitor.check_site(域名)              // 网站状态
        ├→ seo-checker.check_seo(域名)                 // SEO全面检测
        ├→ seo-checker.check_pagespeed(域名)            // 页面速度
        └→ content-tracker.list_content(client_id)      // 内容发布记录

Step 3: 竞品监控（调用 competitor-monitor 技能）
        → 遍历竞品列表
        → fetch(竞品博客页URL) × N
        → 对比分析内容差距

Step 4: 选题生成
        → content-tracker.suggest_content(client_id)    // AI选题建议
        → 结合竞品分析 + CSV数据（如有）
        → 输出2篇博客选题（标题、目标关键词、大纲）

Step 5: 读取本地数据（如有）
        → 读取 客户-XX/数据/ 中的CSV文件
        → 提取流量、排名、转化数据
        → 与上月对比，计算环比

Step 6: 输出与记录
        → 保存到 客户-XX/数据/YYYY-MM_数据分析报告.md
        → 保存到 客户-XX/数据/YYYY-MM_竞品动态简报.md
        → 保存到 客户-XX/数据/YYYY-MM_本月执行计划.md
        → client-manager.add_timeline(id, "完成数据分析与选题输出")
```

**⏸️ 暂停**：输出选题后等确认或调整。

## 数据分析框架

### 新站（无历史数据）

MCP 替代方案：

| 替代数据 | MCP 工具 | 具体做法 |
|----------|----------|----------|
| 页面速度基准 | `seo-checker.check_pagespeed` | 直接检测 |
| SEO 健康度 | `seo-checker.check_seo` | 全面检测 |
| 网站可用性 | `site-monitor.check_site` | 实时检查 |
| 竞品内容 | `fetch(竞品URL)` | 抓取竞品博客页 |
| 选题建议 | `content-tracker.suggest_content` | AI 推荐选题 |

### 有数据后

| 数据信号 | 意味着什么 | 对应动作 |
|----------|------------|----------|
| 跳出率>80% | 内容与搜索意图不匹配 | 重写页面开头 |
| 停留时间<30秒 | 内容质量差或加载慢 | 检查速度，优化深度 |
| 自然流量停滞 | 关键词排名未提升 | 增加内链，更新内容 |
| 转化率<0.5% | CTA不够显眼 | 优化CTA位置 |
| 某页面流量突增 | 关键词开始排名 | 优化该页面，添加内链 |

## GA4/GSC 数据接入

客户每月手动导出 CSV（约10分钟）：

**GA4**：
- 报告 → 流量获取 → 概览 → 日期设为上月 → 导出 CSV
- 报告 → 互动度 → 页面和屏幕 → 导出 CSV

**Search Console**：
- 效果 → 日期设为上月 → 导出 CSV（查询 + 页面）

**文件命名**：`YYYY-MM_GA4_流量概览.csv` / `YYYY-MM_GSC_查询报告.csv`  
**放置位置**：`客户-XX/数据/`

## GSC数据使用规范（必读）

> 来源：客户B实战教训——向客户报"排名第1"，客户搜不到，信任受损。

1. **过滤假关键词**：query 中包含 `-site:` 或引号+排除条件的，必须从报告中剔除
2. **排名用"平均排名"**：position 6.5 不代表稳定在第6名，报告中标注"GSC平均排名"
3. **禁止夸大表述**：禁用"登顶""排名第1""已进入首页"。用"GSC显示平均排名X.X"
4. **区分词类**：长尾词（4+单词）排名预期与核心商业词（1-2词）完全不同，不混谈
5. **客户可验证声明**：报告末尾加注"排名因搜索地区、语言、设备不同而异，以GSC数据为准"

## 异常告警规则

数据分析过程中发现以下情况时，立即标记并触发对应技能：

| 异常信号 | 阈值 | 触发行动 |
|---------|------|---------|
| 某页面跳出率>85% | 环比上升>10% | → **content-refresh** 紧急更新 |
| 某页面流量骤降>50% | 7天 vs 上7天 | → **rank-tracking** 紧急诊断 |
| 核心关键词掉出前20 | 排名下降>10位 | → **daily-growth** 优先响应 |
| 竞品在我方核心词出现 | GSC新增竞品URL | → **competitor-monitor** 深度分析 |
| 0询盘超过14天 | GA4转化事件=0 | → 检查表单/CTA是否正常 |
| 索引率<70% | gsc_index_status | → **tech-optimization** 索引修复 |

## 选题质量检查清单

选题输出前逐项确认：

- [ ] 每个选题都有GSC/竞品数据支撑（不是拍脑袋）
- [ ] 选题不与 content-tracker 中已有内容重复
- [ ] 选题不与 topic-cluster 路线图冲突
- [ ] 每个选题有明确的搜索意图分类（认知/考虑/决策）
- [ ] 关键词难度评估：新站优先长尾（4+词），避免直攻大词
- [ ] 选题数量合理（每月2-4篇，不超出产能）
- [ ] timeline已记录：client-manager.add_timeline(id, "月度分析完成: X篇选题输出")

## 与其他技能联动

| 输入来源 | 联动 |
|---------|------|
| topic-cluster路线图 | 从路线图中选取本月待写文章 |
| competitor-monitor缺口清单 | 优先填补竞品覆盖但我方缺失的主题 |
| rank-tracking机会词 | 排名11-20的词优先写内容冲首页 |
| content-refresh衰退文章 | 需更新的旧文章纳入本月工作量 |

| 输出去向 | 联动 |
|---------|------|
| 选题确认后 | → **content-production** 12阶段执行 |
| 发现异常 | → 对应技能紧急处理（见告警规则） |
