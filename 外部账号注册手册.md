## 任务 1：HARO / Qwoted / Featured 注册（30 分钟）

### 背景

原 HARO 平台 2024-06-30 被 Connectively 收购替代。现在用以下 5 个：

| 平台 | URL | 偏好 | 优先级 |
|---|---|---|---|
| **Connectively** | <https://connectively.us/> | HARO 直接接班，记者多 | ⭐⭐⭐ 必注册 |
| **Qwoted** | <https://www.qwoted.com/> | 专业记者 + 高质量征集 | ⭐⭐⭐ 必注册 |
| **Featured** | <https://featured.com/> | B2B 友好 | ⭐⭐ 推荐 |
| **HelpaB2BWriter** | <https://helpab2bwriter.com/> | B2B 专项 | ⭐⭐ 推荐 |
| **JournoRequests** | <https://journorequests.com/> | X/Twitter 聚合（免费） | ⭐ 可选 |

### 操作步骤

#### Connectively（必做）

1. 访问 <https://connectively.us/>
2. **Sign up as Source/Expert**（不是记者）
3. 填邮箱（建议用 `hkf+pr@yourdomain.com` 这种带 +pr 后缀，便于后期过滤）
4. 设密码 → 验证邮箱
5. 完成 Profile：
   - Industries: Manufacturing / Building Products / Industrial Equipment / Adhesives & Sealants（按 4 客户行业全选）
   - Expertise: 你公司提供的服务类型
   - LinkedIn URL: 必填（增加可信度）
6. 设邮件订阅：**3x daily**（早 / 午 / 晚各一次征集汇总）
7. 把账号密码存到 `~/.claude/.env`：

   ```
   CONNECTIVELY_EMAIL=hkf+pr@yourdomain.com
   CONNECTIVELY_PASSWORD=xxxxx
   ```

#### Qwoted

1. 访问 <https://www.qwoted.com/>
2. **Get Started → As a Source**
3. 填邮箱 / 密码 / 完成 Profile（同上）
4. 选 Topics：B2B / Manufacturing / Sustainability / Construction / Adhesives 等
5. 邮件订阅：daily digest

#### Featured

1. <https://featured.com/>
2. **Become an Expert**
3. 同上注册流程
4. Featured 偏好 listicle / quote round-up 类内容

### 注册完后告诉智能体

在 web-ops 目录对话：

```
我注册了 Connectively / Qwoted / Featured 账号，账号密码已存 ~/.claude/.env
帮我把每天收到的 HARO 征集邮件解析后自动跑 haro-draft-generator
```

智能体会：
1. 配 IMAP 监听这 3 个平台的发件邮箱
2. 解析每日邮件（30-50 条征集）
3. 按客户行业过滤（demo-c: EPS / demo-b: 胶水 / demo-a: 冷链 / Demo-D: 听力保护）
4. 匹配的每条调 `haro-draft-generator.mjs` 出草稿
5. 推企微让你审 → 你点"提交"后智能体自动登录平台提交

### 4 客户 expertise-pack 填空（与平台注册并行）

每个客户找他们的专家（老板 / 技术总监）填一份专家素材，作为 quote 引用基础。

模板见 [.claude/skills/digital-pr-haro.md](.claude/skills/digital-pr-haro.md)。

每客户填空大约 30-60 分钟，关键内容：
- 公司基本信息（建厂年 / 产能 / 主要市场 / 认证）
- 5 个核心数据点（具体数字 + 单位 + 时间）
- 3 个反直觉断言（其他专家答不出的）
- 2-3 个实证案例
- 专家署名 + LinkedIn

填好后存到客户站 `客户/<X>/website/docs/expertise-pack.md`。

## 任务 2：Microsoft Clarity 埋点（1 小时）

### 背景

CRO（转化率优化）需要看真实用户行为：哪里点击、哪里离开、看到什么内容才转化。Microsoft Clarity 是 Hotjar 的免费替代（无流量限制）。

### 操作步骤

1. 访问 <https://clarity.microsoft.com/>
2. **Sign in**（用 Google / Microsoft / Facebook 账号）
3. **+ Add new project**
4. 输入：
   - Project name: `demo-c`
   - Website URL: `https://demo-c.com`
   - Category: B2B / Manufacturing
5. 创建后获得 **tracking code**（一段 JS，类似）：

   ```html
   <script type="text/javascript">
     (function(c,l,a,r,i,t,y){
       c[a]=c[a]||function(){(c[a].q=c[a].q||[]).push(arguments)};
       t=l.createElement(r);t.async=1;t.src="https://www.clarity.ms/tag/"+i;
       y=l.getElementsByTagName(r)[0];y.parentNode.insertBefore(t,y);
     })(window, document, "clarity", "script", "<PROJECT_ID>");
   </script>
   ```

6. **复制这段代码**
7. 切建站智能体（cd 到对应客户站 + 让建站智能体加到 BaseLayout.astro `<head>` 里）：

   ```
   "把 Microsoft Clarity tracking code 加到 demo-c BaseLayout.astro head 里
   <PROJECT_ID> 是 abc123def"
   ```

8. 重复 4 次（4 客户站各一个 project）
9. 等 24 小时数据开始进来

### Clarity 给智能体什么

数据：
- Heatmap（哪里点击多 / 哪里被忽略）
- Session replay（用户操作回放）
- Dead clicks（点了没反应的元素 = 设计问题）
- Rage clicks（用户狂点 = 没找到想要的）
- Scroll depth（看到第几屏离开）

智能体能用来：
- 改 CTA 位置（如果用户都没看到底部）
- 改 H2 顺序（如果中间章节都被跳过）
- 修产品页（如果产品图被疯狂点击但没链接）

### 注册完后告诉智能体

```
4 客户的 Microsoft Clarity 都装好了，project ID 我已存到 ~/.claude/.env
帮我每周分析 Clarity 数据 + 找改进点
```

智能体会写一个 `scripts/clarity-analyzer.mjs` + 集成 daily-cron 周日段。

## 任务 3：Looker Studio Dashboard（半天 + 客户配合）

### 背景

GA4 原生界面查报表慢，Looker Studio 能做客户化的 dashboard（一目了然），月报里直接截图。

### 操作步骤

#### 第 1 步：客户共享 GA4 给你（5 min × 4 客户）

让客户做以下操作（用截图教学最快）：

1. 客户登录 <https://analytics.google.com/>
2. 选他们的 property
3. **Admin（齿轮）→ Property → Property access management**
4. **+ Add users**
5. 输入运营人员邮箱（建议 `contact@example.com` 或专用账号）
6. 权限选 **Analyst**（看数据 + 建受众，不能改 property）
7. **Add**

客户做完，你邮箱会收到通知。

#### 第 2 步：建 Looker Studio dashboard（1h × 4 客户）

1. 访问 <https://lookerstudio.google.com/>
2. **Create → Report**
3. **Add data → Google Analytics 4** → 选客户 property
4. 选模板（最快）：
   - 模板库 <https://lookerstudio.google.com/gallery>
   - 搜 "GA4 Acquisition Overview" 或 "GA4 Marketing Overview"
   - 复制模板 → 修改数据源指向客户 property
5. 自定义 4 个核心 page：
   - **Acquisition**: 流量来源 / Top 10 着陆页 / Channel 分布
   - **Behavior**: 高跳出率页面 / 平均停留 / 滚动深度
   - **Conversion**: 5 转化事件（whatsapp/email/phone/quote/lead）漏斗
   - **AI Citation**: 过滤 referrer 含 perplexity/chatgpt/claude/etc.
6. **Share** → 添加你的邮箱（read 权限）+ 添加客户邮箱（read，让客户能看月报）
7. 复制每个 dashboard URL 存到 `~/.claude/.env`：

   ```
   LOOKER_CHINAEPS=https://lookerstudio.google.com/reporting/xxx
   LOOKER_DESAIIND=https://lookerstudio.google.com/reporting/yyy
   LOOKER_Demo-A=https://lookerstudio.google.com/reporting/zzz
   LOOKER_HEARINGPROTECT=https://lookerstudio.google.com/reporting/www
   ```

#### 第 3 步：告诉智能体

```
4 客户 Looker Studio dashboard 都建好了, URL 已存到 ~/.claude/.env
帮我每月 28 号月报里附上 dashboard 截图链接
```

智能体会：
1. 月报 prompt 里加"贴 Looker URL"
2. attribution-analytics skill 真激活
