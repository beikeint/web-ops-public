## ① Google Search Console · 添加服务账号（最重要！5 分钟）

**为什么必须做**：运营人员每天早上自动巡检会读 GSC 数据看有没有新的错误，没授权时读不到（403 权限不足）。做完这一步，每日巡检就能自动报告"今天哪些词在涨、哪些页面被搜索"。

**步骤**：

1. 打开 https://search.google.com/search-console
2. 选择资源：`{{DOMAIN}}/`
3. 左侧菜单点 **设置** → **用户和权限**
4. 点右上角 **添加用户**
5. 电子邮件粘贴：
   ```
   gsc-reader@example-seo.iam.gserviceaccount.com
   ```
6. 权限选择：**受限**（只读，不会改任何设置）
7. 点 **添加**

**完成后告诉运营人员**：回一句"GSC 服务账号已添加"即可，运营人员会在第二天巡检时自动验证。

> 如果您还没有把网站添加到 GSC，请先告诉运营人员，会协助您完成验证。

## ② Bing Webmaster Tools · 从 GSC 一键导入（5 分钟）

**为什么做**：Bing 在欧美（尤其是企业用户）市场份额约 10-20%，有欧洲客户就值得覆盖。

**最快路径（强烈推荐）**：从 GSC 导入，不用重新验证。

1. 打开 https://www.bing.com/webmasters
2. 用**和 GSC 同一个 Google 账号登录**（右上角 Sign in → Google）
3. 登录后会提示 **Import your sites from Google Search Console** → 点 **Import**
4. 勾选 `{{DOMAIN_BARE}}` → **Import**
5. 几秒后 Bing 会自动完成验证（因为 GSC 已经验证过域名所有权）

**完成后告诉运营人员**："Bing 已从 GSC 导入完成"。

**备用方案**（如果上面第 3 步没看到导入选项）：
- 点 **Add a site manually** → 填 `{{DOMAIN}}` → 选 **Meta tag** 验证方式
- 复制页面上给的 `<meta name="msvalidate.01" content="XXXX">` 里 `content=""` 的那串码
- 发给运营人员，运营人员粘贴到网站后重新部署，回来点 **Verify**

## ③ Yandex Webmaster · 覆盖俄语区买家（10 分钟，需运营人员配合）

**为什么做**：有俄语版网站且目标市场包含俄罗斯/中亚时必做。Yandex 在俄语区搜索市场份额约 60%。

**步骤**：

1. 打开 https://webmaster.yandex.com
2. 需要先注册 Yandex 账号（如果没有）：右上角 **Sign up** → 用邮箱或手机号都行（**建议用工作邮箱 {{TEAM_EMAIL}}**，方便团队共享）
3. 登录后点 **Add site** → 填 `{{DOMAIN}}/`
4. 选择 **Meta tag** 验证方式
5. 复制页面上给的 `<meta name="yandex-verification" content="XXXX">` 里 `content=""` 的那串码
6. **发给运营人员**（微信/邮件都行），运营人员会粘贴到网站部署
7. 运营人员通知您部署完成后，回到 Yandex 页面点 **Verify**

**完成后告诉运营人员**："Yandex 验证码：xxxxxxxx"。

> 如果您的目标市场不包含俄语区，本项可跳过。

## ④ LinkedIn 公司页（10 分钟）

**为什么做**：
- B2B 外贸买家会在 LinkedIn 上查您是不是真公司、有没有团队
- 网站底部的社媒图标点击过去如果是空的，对信任度是减分
- 日后可以作为内容分发渠道（发博客/产品到 LinkedIn）

**步骤**：

1. 打开 https://www.linkedin.com/company/setup/new/
2. 需要您自己有个 LinkedIn 个人账号（没有就先花 5 分钟注册，建议用 {{TEAM_EMAIL}}）
3. 公司信息填写：

| 字段 | 填什么 |
|---|---|
| Company name | `{{COMPANY_EN}}` |
| LinkedIn public URL | 系统会自动生成（建议和公司名接近） |
| Website | `{{DOMAIN}}` |
| Industry | **{{INDUSTRY_EN}}** |
| Company size | `11-50 employees`（或实际规模） |
| Company type | `Privately Held` |
| Logo | 上传公司 logo（线上路径 `{{LOGO_PATH}}`，或提供原始设计稿） |
| Tagline | `{{TAGLINE_EN}}` |

4. 创建完成后，把公司页的 URL 发给运营人员（例如 `https://www.linkedin.com/company/xxx`）
5. 运营人员会在网站 footer 加上 LinkedIn 图标

**完成后告诉运营人员**："LinkedIn 公司页：https://www.linkedin.com/company/xxx"。

## 常见问题

**Q：必须用什么邮箱注册？**
A：Google 系用已有 Google 账号就行；Bing/Yandex/LinkedIn **强烈建议用团队邮箱 {{TEAM_EMAIL}}**，而不是个人邮箱。以后交接、团队协作都方便。

**Q：这些账号会不会收费？**
A：全部免费。LinkedIn 有付费"Premium"功能（招聘/找客户），公司页本身免费。

**Q：我没时间做，能不能运营人员全代做？**
A：GSC 必须您做（服务账号只能站点所有者添加）。Bing 从 GSC 导入也必须您登录 Google 做。Yandex 和 LinkedIn 理论上运营人员能代做，但涉及您的邮箱密码，不推荐。

**Q：做错了怎么办？**
A：以上每一步都可逆（加错用户能删、验证错了能重新验证），不用担心。
