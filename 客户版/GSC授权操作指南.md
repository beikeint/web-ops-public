# Google Search Console 授权操作指南

> 致尊敬的客户：
> 本文档指导您将 Google Search Console（谷歌搜索控制台）的数据权限授权给我们。
> 授权后，我们的运营系统可以**每天自动读取您网站的搜索数据**，实现：
> - 自动检测新增的404错误页面
> - 自动监控关键词排名变化
> - 自动发现索引异常并及时修复
> - 月报数据自动生成，无需您手动导出
>
> **整个过程约需10-15分钟，全部在网页上操作，不需要安装任何软件。**
> **授权的是"只读"性质的权限，我们无法修改您的任何Google数据。**

## 准备工作

在开始之前，请确认：

- [ ] 您有一个 Google 账号（就是您的 Gmail 邮箱）
- [ ] 您的网站已经添加到了 Google Search Console
  - 如果还没有添加，请先告诉我们，我们协助您完成
- [ ] 您能访问 Google 网站（需要科学上网工具）

> **没有 Google 账号？** 请先注册一个：https://accounts.google.com/signup
> **不知道什么是 Search Console？** 它是 Google 提供的免费工具，帮助网站主了解自己的网站在 Google 搜索中的表现。我们帮您注册和配置。

## 第一步：创建 Google Cloud 项目（约3分钟）

### 1.1 打开 Google Cloud 控制台

在浏览器中访问：

```
https://console.cloud.google.com
```

用您的 Google 账号登录。

> 如果是第一次使用，Google 会要求您同意服务条款。勾选同意即可，**不需要绑定信用卡，也不会产生任何费用**。

### 1.2 创建新项目

登录后，看页面顶部：

```
Google Cloud    [项目名称 ▼]    搜索栏...
```

1. 点击顶部的 **项目名称下拉箭头 ▼**
2. 在弹出的窗口中，点击右上角的 **"新建项目"**

填写信息：

| 项目 | 填什么 |
|------|--------|
| 项目名称 | 填您的公司英文简称，例如 `mycompany-seo` |
| 组织 | 保持默认（"无组织"） |
| 位置 | 保持默认 |

3. 点击 **"创建"**

等待几秒钟，页面右上角会出现通知"项目已创建"。

4. 再次点击顶部的 **项目名称下拉箭头 ▼**，选择刚创建的项目，确保切换到新项目

> **怎么确认切换成功了？** 顶部项目名称显示的是您刚填的名称（如 `mycompany-seo`），就对了。

## 第二步：启用 Search Console API（约1分钟）

### 2.1 进入 API 库

在页面左侧，找到菜单：

```
☰ 导航菜单 → API 和服务 → 库
```

> 如果左侧没有菜单，点击页面左上角的 **☰ 三条横线图标**展开。

### 2.2 搜索并启用

1. 在搜索框中输入：`Search Console API`
2. 在搜索结果中，点击 **"Google Search Console API"**
3. 在详情页面，点击蓝色的 **"启用"** 按钮

等待几秒钟，页面会跳转到 API 概览页，说明已启用成功。

> **看到"管理"按钮而不是"启用"？** 说明已经启用过了，直接进入下一步。

---

## 第三步：创建服务账号（约3分钟）

### 3.1 进入凭据页面

在页面左侧菜单：

```
☰ 导航菜单 → API 和服务 → 凭据
```

### 3.2 创建服务账号

1. 点击页面顶部的 **"+ 创建凭据"**
2. 页面会出现 **"凭据类型"** 选择界面：
   - **"您使用的是哪个 API？"** → 选 `Google Search Console API`
   - **"您要访问哪些数据？"** → 选 **"应用数据"**
3. 点击 **"下一步"**

在 **"您的凭据"** 区域填写：

| 项目 | 填什么 |
|------|--------|
| 服务账号名称 | 填 `gsc-reader` |
| 服务账号 ID | 系统会自动填，不用改 |
| 描述 | 可以填"网站运营数据读取"或留空 |

4. 点击 **"完成"**

> **另一种界面：** 如果您看到的不是上面的流程，而是直接让您选"服务账号"，那么填写名称后点"创建并继续"，后面"角色"和"用户访问权限"两步都**直接跳过**（不选不填，点继续/完成）。

### 3.5 记下服务账号邮箱

创建完成后，页面会回到凭据列表。在"服务账号"部分，您会看到刚创建的账号：

```
gsc-reader@您的项目名.iam.gserviceaccount.com
```

**请把这个邮箱地址复制下来，后面要用。** 可以先粘贴到微信聊天框里保存。

---

## 第四步：下载密钥文件（约2分钟）

### 4.1 进入服务账号详情

在凭据页面的"服务账号"列表中：

1. 点击刚创建的服务账号邮箱（蓝色链接）

### 4.2 创建密钥

1. 点击页面顶部的 **"密钥"** 选项卡
2. 点击 **"添加密钥"** → 选择 **"创建新密钥"**
3. 密钥类型选 **"JSON"**
4. 点击 **"创建"**

浏览器会自动下载一个 `.json` 文件，文件名类似：

```
mycompany-seo-a1b2c3d4e5f6.json
```

**这个文件非常重要，请妥善保存。**

> **安全提示：** 这个文件相当于一把钥匙。不要发到公开群里，通过微信私聊发给我们的运营对接人即可。我们收到后会安全存储，且只用于读取您的搜索数据。

## 第五步：在 Search Console 中授权（约2分钟）

### 5.1 打开 Search Console

在浏览器中访问：

```
https://search.google.com/search-console
```

### 5.2 选择您的网站

如果您有多个网站，在左上角的下拉菜单中选择需要授权的网站（例如 `demo-c.com`）。

### 5.3 添加用户

1. 点击左下角的 **"设置"**（齿轮图标）
2. 点击 **"用户和权限"**
3. 点击右上角的 **"添加用户"**

填写信息：

| 项目 | 填什么 |
|------|--------|
| 电子邮件地址 | 粘贴第三步记下的服务账号邮箱（`gsc-reader@xxx.iam.gserviceaccount.com`） |
| 权限 | 选择 **"完整"** |

4. 点击 **"添加"**

> **为什么选"完整"权限？** "完整"是指可以查看所有数据，但不包括删除网站或修改所有者的权限。我们需要读取索引状态和搜索数据，"受限"权限无法获取完整的索引报告。

### 5.4 确认添加成功

添加后，在用户列表中应该能看到：

```
gsc-reader@xxx.iam.gserviceaccount.com    完整
```

## 常见问题

### Q1：这样做安全吗？会不会影响我的网站？

完全安全。服务账号只有**读取**数据的权限，无法修改您的网站内容、删除数据或更改任何设置。您可以随时在 Search Console 的"用户和权限"中移除这个账号。

### Q2：会产生费用吗？

不会。Google Cloud 项目创建是免费的，Search Console API 的调用量在免费额度之内，不会产生任何费用。

### Q3：我有多个网站，需要重复操作吗？

不需要全部重复。只需要重复**第五步**：在每个网站的 Search Console 中添加同一个服务账号邮箱即可。第一步到第四步只需要做一次。

### Q4：我忘记了服务账号邮箱怎么办？

打开 https://console.cloud.google.com → API 和服务 → 凭据 → 服务账号列表中可以找到。

### Q5：密钥文件丢了怎么办？

可以重新创建：进入服务账号详情 → 密钥 → 添加密钥 → 创建新密钥 → JSON。旧密钥建议删除。

### Q6：以后不想授权了怎么取消？

打开 Search Console → 设置 → 用户和权限 → 找到服务账号邮箱 → 点击右侧的删除按钮。即时生效。
