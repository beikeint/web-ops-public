# Path C 客户月度内容深度合作通知模板

> v11.0+1 (2026-05-07 立) — 用于每月 1-3 号给 4 客户老板发"月度深度内容合作通知"
> 落客户简报到 `客户/<客户>/客户简报/<月份>-Path-C-资源征集.md` + 转发客户群
> 不是恳求 — 是产品要求。客户不给资源 = 本月跳深度博客（不强求）

## 通用模板（中文，给客户老板）

```markdown

{{客户老板尊称}} 您好，

为达成行业前 1% 的内容质量目标（每月 1 篇可代表贵公司专业实力的深度博客），
本月我们需要您配合提供 4 类资源：

## 1️⃣ 真运营数据点（≥ 3 个）

请提供贵公司近期生产 / 销售 / 测试系统中的 3 个具体数字。**例**：
- 「{{2025 年我们生产 X 万件 Y 产品}}」
- 「{{单条产线月产能 X 件 / 投资回收 X 月}}」
- 「{{产品 X 的 Y 测试合格率 X.X%（基于 X 次抽样）}}」

→ 用途：嵌入博客每段，让搜索引擎和读者看到真账。

## 2️⃣ 真实客户案例（1 个）

请提供 1 个您愿意公开（可匿名为「中东某 X 厂」）的下游买家成功故事，含：
- 国家 + 行业 + 公司规模
- 应用场景（用了什么产品 / 做什么）
- 实际收益（产能 +X% / 成本 -¥Y / 投资回收 N 月）
- 30-50 字客户反馈 quote
- 时间（具体月份）

→ 用途：博客中专门一节"Real-world Case"，比任何营销文案更有说服力。

## 3️⃣ 内部技术专家配合（1 位）

请指定贵公司 1 位真名工程师 / 总监配合：
- 真名 + 职位（如「张工，R&D 总监」）
- LinkedIn 个人页 URL（必含 `/in/` 路径）
- 一句技术 quote（30-50 字，关于本月主题的专业见解）
- 头像照片（可选，方形 200x200px+）

→ 用途：作为博客作者署名 + 嵌入专家 quote 段，建立 E-E-A-T（Google 信任度的核心）。

## 4️⃣ 原创视觉资产（≥ 3 张）

请提供 3 张以上原创图片：
- 工厂实拍（设备 / 产线 / 生产场景）
- 产品对比图
- 流程图 / 数据图表

**禁止**：网络下载图 / 库存图 / 同行图。
→ 用途：博客内文配图，让"真实感"压过竞品的模板感。

## 常见问题

**Q：必须每月都给吗？**
A：是的。这是 Path C 顶级内容服务的产品要求。如果连续 3 月无法配合，
   我们会跟您讨论调整服务模式（基础包 vs 顶级包）。

**Q：数据公开有风险吗？**
A：所有客户案例可匿名（"中东某 EPS 厂"），数据可粗化（"年产 380 万对"vs精确数）。
   核心是"真"，不是"精确到小数点"。

**Q：内部专家不愿出镜怎么办？**
A：LinkedIn 个人页是必须的（建立 E-E-A-T），头像照可选。专家可以是您本人 / 老员工 / 
   外部行业顾问。

**Q：我能给一份资源用 3 个月吗？**
A：不能。Google 越来越判定"持续真实创作"vs"老资源反复用"。每月新数据 / 新案例。
```

## 派发流程（运营人员每月 1-3 号执行）

1. **打开本模板**
2. **填 4 客户实际数据**（公司名 / 老板尊称 / 例数据具体化）
3. **群发**到 4 客户老板个人微信 / 客户群
4. **跟踪**：5/7 号前未回复 → 单独追问 1 次 / 还不回 = 本月跳过
5. **回复后**：把客户给的 4 类资源放到 `客户/<X>/website/docs/path-c-resources/2026-05/`
6. **告诉我**："{{客户}} 5 月资源齐全，启动深度博客"
7. **我自动跑** deep-content-production 18 阶段 7-10 天交付

## 反元规则（防止派发漂移）

- **不要降级语气恳求**：这是产品要求。客户拒绝就跳月，不缠不催不跪舔
- **不要凑数启动**：4 类资源缺任一 = 不启动，不允许"用 3 类先写"
- **不要月内多次催**：发 1 次通知 + 5/7 截止前 1 次提醒 = 上限 2 次
- **不要承诺"询盘 +X%"**：90+ 内容是高概率高质量询盘，但不是 SLA
