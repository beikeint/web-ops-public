## v9.2 · pm2 + deployer 修复 + memory 持久化 + MCP 扩展路线图（2026-04-15）

### 5 件事

- **修复 pm2 自主巡检**：stderr 捕获 + 部分结果容错（之前巡检失败就整个中断）
- **修复 deployer MCP 密码作用域 bug**：多客户切换时密码串号
- **建立 memory 持久化记忆系统**：6 个记忆文件，客户档案持久化
- **完善竞品扫描 MCP 调用链**：fetch → 解析 → 入库流程打通
- **新增 MCP 扩展路线图**：email / linkedin / calendar / apollo / wechat 共 5 个 MCP 激活计划

### Why

前三项是 v9.1 实战中暴露的稳定性问题。MCP 扩展路线图是为"做到 30 客户规模"做基础设施准备。

### How to apply

- 新客户接入时：pm2 cron 应该配置 stderr 日志，报错不阻塞其他客户
- 多客户 deployer：每次用前重置密码作用域
- memory 持久化：见 `memory/` 目录的 6 个记忆文件

## v9.1 · P0-P4 实战能力补全（2026-04-12）

### 事实

为客户B（Demo-C / Demo-C）完成 **P0-P4 全部硬件升级**，5 层能力全部落地。成果**可复用到所有客户站点**。建站智能体和网站运营-web-ops共享此基础设施 — 建站阶段部署 P0-P1，运营阶段启用 P2-P4。

### P0 · 转化追踪闭环

- 全站 **5 个 GA4 事件**：`whatsapp_click` · `email_click` · `phone_click` · `quote_click` · `exit_intent generate_lead`
- 事件委托在 `BaseLayout` GA4 脚本中实现 — **一次部署全站生效**
- **新客户建站时必须包含**（已加入 QA 必检清单第 7 项）

### P1 · 索引加速引擎

- IndexNow API key 部署到 `public/` 目录
- `scripts/index-boost.mjs` 批量提交 URL 到 Bing / Yandex
- 每次发布新内容后运行 + GSC Sitemap 重提交
- **已加入 QA 必检清单第 8 项**

### P2 · 智能选题引擎

- **方法**：GSC 28 天数据（query + page 双维度） + 竞品 10 关键词 × 5 竞品网站
- **输出**：内容缺口清单（搜索量 × 商业价值排序），融入 `data-analysis` 技能
- **Demo-C实战**：发现 3 个关键缺口 — 鱼箱机 / 模具设计 / 三明治板，新站 40% 页面未索引问题同步解决

### P3 · 多渠道询盘入口

- **ROI 计算器**页面（`/en/roi-calculator/`）：产能参数 → 回本周期 → 留联系方式
- GA4 事件：`roi_calculated`（参与） + `generate_lead`（转化）
- **模板可复用到其他 B2B 客户站**

### P4 · 自主运营

- `pm2 cron` 每天 **08:00** 触发 `daily-cron.mjs`
- Claude CLI 自动跑所有客户巡检
- 结果**推送企业微信内部群**（Webhook）+ 缓存到 `reports/latest.txt`，快捷指令秒回

### 其他补齐

- **强化自我进化机制**（全局 `feedback_doc_sync.md` 规则产生于此次迭代）

## v9.0 · search-analytics MCP（2026-04-10 前后）

### 事实

新增 `search-analytics` MCP，GSC 数据全自动化。巡检 / 排名追踪 / 月报**不再依赖客户手动导出 CSV**。

### Why

v8.0 的"数据分析"技能依赖客户每次手动导出 GSC 数据，卡点严重。v9.0 直接接入 GSC API，实现完全自动化。

### How to apply

- 新客户接入：`client-onboarding` Step 5 增加 GSC/GA4 数据接入步骤（带 OAuth 授权）
- `analytics-api` 技能现在直接用 MCP 工具而不是读 CSV（对比表在技能文件里）
- `weekly-check` 的索引健康（20 分）用 GSC 真实数据打分
