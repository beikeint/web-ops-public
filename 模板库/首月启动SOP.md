# 首月启动SOP — 新客户网站上线执行手册

> 适用场景：网站运营智能体建站客户网站上线当天及首月  
> 版本：v1.0 | 创建日期：2026年4月  
> 执行人：运营人员

## 一、网站上线当天执行清单

### 1.1 GA4配置（预计30分钟）

**步骤1：创建GA4媒体资源**

1. 登录 analytics.google.com
2. 左下角「管理」→「+ 创建媒体资源」
3. 填写：
   - 媒体资源名称：`[客户公司英文名] - Website`
   - 时区：选客户主要目标市场时区（北美客户选美东，全球客户选UTC）
   - 货币：USD
4. 选择行业类别和公司规模
5. 创建完成，记录 **Property ID**（G-XXXXXXXXXX）

**步骤2：安装追踪代码**

```html
<!-- 将以下代码添加到网站 <head> 标签内 -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
```

确认方式：
- 打开网站 → 浏览器F12 → Network → 搜索 `gtag` → 有请求即成功
- 或 GA4 后台「实时」报告能看到自己的访问

**步骤3：配置数据流**

1. 管理 → 数据流 → 选择网站数据流
2. 开启「增强型衡量」，确保以下全部打开：
   - ✅ 网页浏览
   - ✅ 滚动
   - ✅ 出站点击
   - ✅ 网站搜索
   - ✅ 表单互动
   - ✅ 文件下载

**步骤4：配置转化事件**

1. 管理 → 事件 → 创建事件
2. 必配的转化事件：

| 事件名 | 触发条件 | 用途 |
|--------|----------|------|
| `form_submit` | 表单提交成功 | 追踪询盘 |
| `click_email` | 点击mailto链接 | 追踪邮件点击 |
| `click_whatsapp` | 点击WhatsApp链接 | 追踪WhatsApp点击 |
| `click_phone` | 点击tel链接 | 追踪电话点击 |

3. 将以上事件标记为「转化」

**步骤5：关联Search Console**

1. 管理 → 产品关联 → Search Console关联
2. 选择对应的Search Console媒体资源
3. 确认关联

**完成后立即做**：
- [ ] 将 GA4 Property ID 填入客户档案
- [ ] 截图保存配置完成状态
- [ ] 用自己手机访问网站，确认GA4实时报告能看到数据

### 1.2 Search Console验证和Sitemap提交（预计20分钟）

**步骤1：添加网站资源**

1. 登录 search.google.com/search-console
2. 「添加资源」→ 选择「网域资源」→ 输入域名（不带https://）
3. 选择验证方式（推荐顺序）：
   - **首选**：DNS验证（最彻底，覆盖所有子域名）
   - 备选：HTML文件上传
   - 备选：HTML标签（meta tag）

**DNS验证步骤**：
1. 复制Google提供的TXT记录值
2. 登录域名DNS管理面板
3. 添加TXT记录：
   - 主机记录：`@`
   - 记录类型：`TXT`
   - 记录值：粘贴Google提供的值
   - TTL：默认
4. 回到Search Console点击「验证」
5. DNS传播可能需要几分钟到几小时，失败的话稍后重试

**步骤2：提交Sitemap**

1. 左侧菜单 →「站点地图」
2. 输入sitemap URL：
   - 主站：`sitemap.xml` 或 `sitemap_index.xml`
   - 如果是多语言站，确认sitemap包含所有语言版本的URL
3. 点击「提交」
4. 状态应显示「成功」

**步骤3：请求索引**

1. 顶部搜索框输入首页URL → 「请求编入索引」
2. 对以下重要页面也执行请求索引：
   - 首页
   - 主要产品分类页（2-3个）
   - 联系页面
   - 关于我们
3. 注意：每日请求索引有次数限制（约10-20次），优先处理最重要的页面

**步骤4：基础检查**

- [ ] 「网址检查」工具测试首页 → 确认可索引
- [ ] 检查「覆盖范围/页面」报告 → 记录初始状态
- [ ] 检查「移动设备易用性」→ 确认无错误
- [ ] 检查robots.txt是否正确（`域名/robots.txt`）

**完成后立即做**：
- [ ] 将验证方式和Sitemap URL填入客户档案
- [ ] 截图保存Sitemap提交成功状态

### 1.3 Web3Forms表单测试（预计15分钟）

**步骤1：确认表单配置**

1. 打开网站所有含表单的页面，逐一检查：

| 检查项 | 方法 | 预期结果 |
|--------|------|----------|
| 表单是否显示 | 目测 | 表单完整显示，无布局错乱 |
| 必填字段标识 | 目测 | 必填项有 * 标识 |
| Access Key | 查看源码或配置 | Key已正确填入 |
| 通知邮箱 | Web3Forms后台 | 已配置正确的收件邮箱 |

**步骤2：提交测试**

对每个表单执行一次完整测试提交：

```
测试数据：
- Name: Test Submission [日期]

- Company: Backflip Test

```

逐一确认：
- [ ] 提交后显示成功提示/感谢页面
- [ ] 通知邮箱收到提交内容
- [ ] 邮件内容格式正确、字段完整
- [ ] 如有CC邮箱，CC也能收到

**步骤3：多语言表单测试**

如果是多语言站点，至少测试：
- [ ] 英文版表单
- [ ] 另外1个语言版本的表单（确认多语言表单都指向正确的Access Key）

**步骤4：记录表单信息**

将以下信息填入客户档案：
- 表单工具名称和版本
- Web3Forms Access Key

## 二、首月无历史数据的处理方式

### 2.1 数据字段分类

新网站上线首月，各类数据的可用性不同。按以下规则处理：

#### 🔴 填"基准建立中"的字段（需要1-3个月积累）

这些数据在首月没有历史对比基准，无法做环比分析：

| 字段 | 原因 | 预计可用时间 |
|------|------|-------------|
| 流量环比变化 | 无上月数据可对比 | 第2个月 |
| 关键词排名变化 | 无上月排名可对比 | 第2个月 |
| 转化率趋势 | 样本量不足，转化率不稳定 | 第2-3个月 |
| 流量来源占比趋势 | 首月来源结构不稳定 | 第2-3个月 |
| 竞品排名对比变化 | 无基准对比 | 第2个月 |
| 询盘质量评估 | 样本量不足 | 第3个月 |
| 用户行为漏斗 | 数据量不足以画漏斗 | 第2-3个月 |

**在月报中的处理方式**：
- 环比列填写「—（首月基准）」
- 不强行分析趋势，改为描述当前状态
- 备注"本数据将作为后续月份的对比基准"

#### 🟡 有数据但需谨慎解读的字段（首月数据波动大）

| 字段 | 首月特点 | 处理方式 |
|------|----------|----------|
| 总UV/PV | 数值小，波动大 | 如实记录，不做过度解读 |
| 跳出率 | 样本量小，不具统计意义 | 记录但标注"参考值" |
| 平均停留时间 | 同上 | 记录但标注"参考值" |
| 关键词排名 | 新站排名不稳定，波动剧烈 | 记录快照，但说明"新站排名通常需要2-3个月稳定" |
| Search Console数据 | 展示和点击数极低 | 如实记录，重点关注"被索引了多少页面" |

**在月报中的处理方式**：
- 如实记录数值
- 加注释说明首月数据的局限性
- 重点放在"索引进度"和"基础设施搭建"上

#### 🟢 当月就可以获取的完整数据

| 字段 | 说明 | 采集方式 |
|------|------|----------|
| PageSpeed评分 | 上线当天即可测 | PageSpeed Insights |
| Core Web Vitals | 上线当天即可测 | PageSpeed Insights（实验室数据） |
| SSL证书状态 | 上线当天即可查 | SSL检查工具 |
| 移动端友好度 | 上线当天即可测 | Mobile-Friendly Test |
| 页面总数 | 上线当天即可数 | Sitemap |
| Meta标签完整性 | 上线当天即可查 | 逐页检查 |
| Schema标记 | 上线当天即可查 | Schema验证工具 |
| 404错误 | 上线即可检测 | 爬虫工具或手动 |
| robots.txt | 上线即可查看 | 直接访问 |
| 内链结构 | 上线即可分析 | 爬虫工具 |
| 表单功能 | 上线即可测试 | 手动测试 |
| 索引页面数 | 上线1-2周后可查 | site:域名 搜索 |
| 竞品网站快照 | 随时可做 | 手动分析 |

### 2.2 首月月报的特殊处理

首月月报的重心与常规月报不同：

| 常规月报重点 | 首月月报重点 |
|-------------|-------------|
| 数据环比分析 | 基准数据建立 |
| 流量增长 | 索引进度追踪 |
| 关键词排名变化 | 初始关键词发现 |
| 转化率优化 | 基础设施搭建完成度 |
| 竞品排名对比 | 竞品初始调研 |

**首月月报建议结构调整**：

```
第一章：基础设施搭建报告（替代"数据总览"）
  - GA4配置完成情况
  - Search Console验证和索引进度
  - 表单测试结果
  - 网站健康基准数据

第二章：初始SEO审计（替代"完成工作"）
  - 页面结构和Meta标签检查
  - 技术SEO体检结果
  - 已发现的问题和已修复项

第三章：初始数据快照（替代"数据洞察"）
  - 上线至今的流量概况（即使很少）
  - 首批被索引的页面
  - Search Console初始数据
  - PageSpeed和CWV基准

第四章：竞品初始调研（替代"问题与建议"）
  - 竞品网站概览
  - 竞品内容和SEO策略初判
  - 我方 vs 竞品的差距分析

第五章：第2个月计划（替代"下月计划"）
  - 第一批博客选题
  - 重点攻坚的关键词
  - 待补充的配置项
```

### 2.3 客户档案首月填写指南

首月填写客户档案时，按以下规则：

| 档案部分 | 填写方式 |
|----------|----------|
| 一、基本信息 | 全部填写，不确定的问客户 |
| 二、数据接入 | 上线当天配置完就填，ID/Key等全部记录 |
| 三、竞品信息 | 首月内完成初步调研，至少填3个竞品 |
| 四、关键词配置 | 首月先填"行业常识词"5-10个，第2月根据GSC数据调整 |
| 五、服务配置 | 签约时即填写 |
| 六、基准数据 | 上线当天填健康数据，月底填流量数据 |
| 七、运营备注 | 持续记录 |

### 3.6 指令执行规则

不管哪个指令，你都遵守以下规则：

1. **先读档案**：每次执行前先读取对应客户的 `客户档案.md`，确认最新信息
2. **先输出计划**：告诉我你准备做什么，等我说"开始"或"确认"再执行
3. **结果先存草稿**：所有输出物先保存为草稿文件，等我审核
4. **主动提问题**：如果发现异常或缺失信息，直接告诉我，不要猜
5. **更新档案**：每次执行完，将新获取的信息更新到客户档案
