## 一、每周检查指令

### 指令1：周一流量检查

**使用时机**：每周一上午  
**耗时**：5分钟  
**预期输出**：一行状态记录，异常时附详细报告

```
周一检查

请对当前客户执行周一流量检查：

1. 网站可访问性：
   - 访问首页，确认加载正常（记录响应时间）
   - 访问最新博客文章，确认可访问
   - 检查SSL证书状态

2. Google索引状态：
   - 搜索 site:{{域名}}，记录索引页面数
   - 与上次记录对比，标注变化

3. 流量异常检测（如有GA4数据）：
   - 上周UV是否有异常波动（>30%变化需报告）
   - 自然搜索流量是否正常
   - 是否有页面流量突降（可能被降权）

4. 输出格式：
   [日期] 周一检查：✅正常 / ⚠️异常（描述）
   - 索引页面：XX页（上次XX页）
   - 网站加载：正常/异常
   
   如有异常，详细说明原因和建议行动。

保存到：客户工作目录/优化/本月_周检查记录.md（追加）
```

---

### 指令2：周三询盘检查

**使用时机**：每周三  
**耗时**：5分钟  
**预期输出**：表单状态+本周询盘记录

```
周三检查

请对当前客户执行周三询盘与转化检查：

1. 表单功能验证：
   - 访问联系页面/询价页面，确认表单显示正常
   - 检查表单必填字段、提交按钮是否正常
   - 如果可以，检查Web3Forms服务状态

2. 转化事件检查（如有GA4数据）：
   - 本周form_submit事件触发次数
   - 本周email_click事件触发次数
   - 本周whatsapp_click事件触发次数
   - 各事件是否正常（对比上周）

3. 询盘记录：
   如有新的询盘信息（客户反馈或Web3Forms通知），
   记录到询盘追踪表：
   - 日期、买家公司、买家国家、来源页面
   - 产品类型、询盘内容摘要、质量评级（A/B/C）

4. 异常检测：
   - 如果连续3天无询盘（对于有历史数据的客户）→标记需排查
   - 如果表单异常→立即报告

输出格式：
[日期] 周三检查：表单✅ / 本周询盘X条 / 转化事件正常

保存到：客户工作目录/优化/本月_周检查记录.md（追加）
```

### 指令3：周五竞品监控

**使用时机**：每周五  
**耗时**：5分钟  
**预期输出**：竞品动态摘要

```
周五检查

请对当前客户执行周五竞品与内容检查：

1. 竞品快速扫描：
   从 client-manager 读取竞品清单，访问TOP3竞品的博客页面：
   - 是否有本周新发布的文章（标题+日期）
   - 首页是否有明显改动
   - 如有重要更新，标记为月度分析重点

2. 我方内容索引检查：
   - 检查最近发布的博客文章是否已被Google索引
     （搜索 site:域名/博客URL）
   - 如未索引，在GSC中手动请求索引

3. AI搜索引用检查（GEO）：
   - 搜索我方3个核心关键词（从 client-manager 读取）
   - 查看Google AI概览/SGE中是否引用了我方内容
   - 如被引用→记录是哪篇文章被引用
   - 如未被引用→标记该关键词对应文章需要GEO优化

输出格式：
[日期] 周五检查：
- 竞品：无更新 / [竞品名]发布[文章名]
- 索引：最新文章已索引/未索引
- AI引用：被引用于[关键词] / 未发现引用

保存到：客户工作目录/优化/本月_周检查记录.md（追加）
```

## 二、每月内容指令

### 指令4：博客文章生成（含19项质量检查）

**使用时机**：选题确认后  
**耗时**：每篇2-4小时（含翻译+构建）  
**预期输出**：完整多语言博客+构建产物

```
写博客：[主题名称]

请按以下标准为当前客户撰写博客文章：

一、从 client-manager.get_client 读取：
- 品牌名、域名、产品信息
- 目标语言列表
- 网站代码路径
- 已有博客列表（用于内链规划）

二、撰写英文原版（2000+词）：
- 从买家真实痛点切入，禁止通用模板式开头
- 包含客户真实数据（产能、认证、出口国家数等）

三、19项质量检查（每项必须通过）：

SEO检查（7项）：
1. [ ] Title含目标关键词，60字符以内
2. [ ] Meta Description 155字符以内，含关键词，有吸引力
3. [ ] 前100词出现目标关键词
4. [ ] H2/H3自然包含关键词变体
5. [ ] 内链3-5个（产品页+相关博客）
6. [ ] 图片Alt标签建议（如有图片）
7. [ ] URL slug简洁、含关键词

GEO检查（5项）：
8. [ ] 开头BLUF段落（直接给出核心答案）
9. [ ] 关键结论用表格或列表呈现
10. [ ] 数据有来源标注
11. [ ] FAQ至少7个问题，每个200词以上
12. [ ] 文章结构可被AI直接引用回答问题

内容深度检查（4项）：
13. [ ] 至少3个数据表格或对比表
14. [ ] 至少5个具体数字或统计数据
15. [ ] 至少1个实际案例或场景描述
16. [ ] 包含竞品无法复制的专属信息

技术检查（3项）：
17. [ ] 写入blog-posts.ts成功
18. [ ] npm run build构建成功
19. [ ] 所有语言版本页面存在于dist中

四、翻译为所有目标语言
五、写入网站代码
六、构建并验证

输出：
1. 英文草稿存档（客户工作目录/内容/）
2. 构建产物（dist/）
3. 19项质量检查清单结果
```

### 指令5：页面优化

**使用时机**：有GSC数据后，每月执行一次  
**耗时**：1-2小时  
**预期输出**：优化方案+执行记录

```
页面优化

请基于Search Console数据为当前客户执行页面优化：

1. 读取本月GSC数据（如有CSV），找出：
   a. 高展示低点击的页面（展示>100, CTR<2%）
      → 这些页面需要优化Title和Description
   b. 排名11-20位的关键词
      → 这些关键词有机会冲进首页，需要内链支撑

2. 对每个需要优化的页面：
   - 当前Title → 建议新Title（含关键词，60字符）
   - 当前Description → 建议新Description（155字符，有号召力）
   - 需要从哪些页面添加内链指向此页面
   - 需要添加的FAQ内容（基于搜索查询）

3. 如果优化涉及代码修改：
   - 修改对应的Astro组件或数据文件
   - 重新构建
   - 记录所有修改

输出：
- 客户工作目录/优化/本月_页面优化方案.md
- 修改后的构建产物（如有代码修改）
```

### 指令6：FAQ更新

**使用时机**：每月执行一次（与指令5同步）  
**耗时**：1小时  
**预期输出**：新增FAQ列表+Schema标记

```
FAQ更新

请为当前客户更新FAQ内容：

1. 数据来源（按优先级）：
   a. Search Console中的疑问句式查询词
      （what/how/why/which/when/where/can/do/is开头的词）
   b. Google搜索核心关键词时出现的"People Also Ask"
   c. 竞品FAQ中我们缺失的问题
   d. 客户反馈的高频询盘问题

2. 对每个新增FAQ：
   - 问题（英文）
   - 答案（200-300词，直接回答，含数据）
   - 目标关键词
   - 放置位置建议（哪个页面/博客）
   - FAQ Schema标记代码

3. 质量要求：
   - 答案第一句话直接回答问题
   - 包含相关关键词（自然嵌入）
   - 结尾引导到相关产品/服务页面
   - 每个答案至少200词

4. 翻译为所有目标语言

输出：
- 新增FAQ列表（含Schema代码）
- 建议的放置位置
- 翻译版本
```

## 三、每月技术指令

### 指令8：技术优化（含内链）

**使用时机**：博客发布后，每月一次  
**耗时**：2-3小时  
**预期输出**：优化记录+构建产物

```
技术优化

请对当前客户执行月度技术优化：

一、速度检测
- 对首页、主要产品页、联系页、最新博客执行PageSpeed检测
- 记录移动端和桌面端评分
- 标注不达标项（LCP>2.5s, CLS>0.1, INP>200ms）
- 给出具体优化建议

二、404检测
- 检查sitemap中所有URL是否可正常访问
- 检查GSC中的"页面"报告是否有新的404
- 对发现的404给出处理建议（301重定向/修复/忽略）

三、内链优化
- 为本月新发布的博客文章规划内链：
  a. 从哪些旧文章链接到新文章（锚文本建议）
  b. 新文章应链接到哪些产品页
  c. 新文章应链接到哪些相关博客
- 检测孤岛页面（无内链指向的页面）
- 输出内链执行清单：| 源页面 | 锚文本 | 目标页面 |

四、结构化数据审计（GEO）
- Organization Schema是否最新
- 所有产品页是否有Product Schema
- 所有博客是否有Article + FAQ Schema
- 面包屑BreadcrumbList Schema是否正确

五、如有代码修改，重新构建并验证

输出：
- 客户工作目录/优化/本月_技术优化记录.md
- 客户工作目录/优化/本月_内链优化记录.md
- 构建产物（如有修改）
```

## 四、每月汇报指令

### 指令9：月报生成

**使用时机**：每月22日  
**耗时**：2-3小时  
**预期输出**：完整月度报告

```
生成月报

请为当前客户生成本月月度运营报告：

一、读取数据：
- 客户工作目录/数据/ 下的所有CSV和分析报告
- 客户工作目录/内容/ 下的博客草稿记录
- 客户工作目录/优化/ 下的技术优化和周检查记录
- 客户工作目录/数据/ 下的询盘追踪表（如有）

二、判断使用哪个模板：
- 如果是首月（无上月数据）→ 使用「首月运营报告模板.md」
- 如果有历史数据 → 使用「月度报告模板.md」（三层结构）

三、生成报告：
第一层（30秒速览）：
- 3个核心数字（UV、询盘、关键词排名）
- 一句话结论
- 需要客户关注的事项

第二层（详细数据）：
- 流量数据（含环比、来源、地域）
- 关键词排名变化TOP10
- 询盘数据（来源分析、国家分布）
- 竞品动态
- 本月完成工作清单
- 数据洞察（深度分析）

第三层（行动计划）：
- 问题与建议（按优先级）
- 下月内容计划（选题+关键词+依据）
- 下月技术优化计划
- 下月目标预设
- 需要客户配合的事项

四、质量自检：
- [ ] 所有数字与原始数据一致
- [ ] 环比计算正确
- [ ] 客户名和域名正确
- [ ] 每个建议有数据支撑
- [ ] 无"可能""也许""大概"等模糊用词
- [ ] 下月计划具体可行

输出：客户工作目录/报告/本月_客户名_月度运营报告.md
```

## 五、季度分析指令

### 指令10：季度关键词分析

**使用时机**：每季度末（3/6/9/12月）  
**耗时**：3-4小时  
**预期输出**：关键词矩阵更新+下季度内容计划

```
季度关键词分析

请对当前客户执行季度关键词深度分析：

一、现有关键词评估：
从 client-manager 读取关键词矩阵，逐一检查：
- 当前Google排名（GSC数据或手动搜索）
- 本季度排名变化趋势
- 该关键词带来的点击和询盘
- 是否值得继续投入

分类：
- 上升词（继续强化）
- 停滞词（需要新策略）
- 无望词（考虑放弃或降优先级）

二、新关键词发现：
- GSC中新出现的有价值查询词
- 竞品新覆盖但我们没有的关键词
- Google自动补全和People Also Ask中的新词
- 行业趋势带来的新搜索需求

三、关键词矩阵更新：
- 升级为P0的词（表现超预期或商业价值更高）
- 降级或移除的词（竞争过高或意图不匹配）
- 新增的词（基于发现）

四、下季度内容计划：
基于更新后的关键词矩阵，推荐下季度6篇文章选题：
- 每篇含：标题、目标关键词、搜索意图、对应集群

输出：
- 客户工作目录/数据/本季度_关键词分析报告.md
- 通过 client-manager.update_client 更新关键词矩阵和内容路线图
```

### 指令11：季度竞品深度分析

**使用时机**：每季度末（与指令10同步）  
**耗时**：3-4小时  
**预期输出**：竞品深度报告+策略建议

```
季度竞品深度分析

请对当前客户的竞品执行季度深度分析：

一、竞品内容盘点：
从 client-manager 读取竞品清单，对每个竞品：
- 访问博客页面，统计当前文章总数
- 与上季度对比，计算新增文章数
- 分析新增文章的主题方向
- 评估内容质量（字数、深度、结构化）

二、竞品排名对比：
- 在核心关键词上，我方vs各竞品的排名对比
- 本季度排名此消彼长的变化
- 竞品新增排名的关键词（我方没有的）

三、竞品网站变化：
- 首页/导航是否改版
- 是否有新产品/新页面
- 是否增加了多语言版本
- 技术栈是否有变化

四、威胁与机会评估：
- 最大竞争威胁是什么
- 最大的差异化机会是什么
- 竞品做了什么我们应该跟进的
- 竞品没做什么是我们的独特优势

五、策略建议：
- 下季度应对策略
- 需要优先填补的内容缺口
- 可以利用的竞品弱点

输出：
- 客户工作目录/数据/本季度_竞品深度分析.md
- 通过 client-manager.update_client 更新竞品信息
```
