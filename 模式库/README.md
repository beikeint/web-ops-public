# 网站运营-web-ops · 模式库

> **作用**：跨客户经验自动复用，让第 100 个客户不重复踩前 99 个客户的坑
> **建立时间**：2026-04-27（v10 产品级路线图第一批基础设施）

---

## 目录结构

```
模式库/
├── README.md                  ← 本文件
├── 成功模式/                  ← 真实跑通的成功打法（含数据证明）
├── 失败教训/                  ← 踩过的坑（含根因 + 防止再次踩坑的检查项）
└── 行业蓝图/                  ← 按行业总结的运营 SOP（B2B 工业 / 跨境 B2C / 等）
```

## 命名规范

### 成功模式
`<动作>-<结果>.md` 例：`多语种沙盒期+延伸效应.md`

### 失败教训
`<问题>-<影响>.md` 例：`旧WP-sitemap-未清-展示量锁死.md`

### 行业蓝图
`<行业类型>.md` 例：`B2B工业化工.md`

---

## 自动应用机制（产品级目标，渐进实现）

**Phase 1（当前）**：人工查阅
- 新客户接入时由智能体读"行业蓝图"对应文件
- 巡检中发现已知问题 → 智能体先查"失败教训库"

**Phase 2（产品级）**：智能体自动应用
- `/接入客户` 命令自动加载对应行业蓝图
- 出现已知问题 → 自动应用对应失败教训库的修复脚本
- 月度 review 自动沉淀新成功 / 新失败到模式库
