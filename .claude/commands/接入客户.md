激活网站运营智能体 v10，**5 分钟把新客户 $1 接入 web-ops 完整运营体系**。

## 🎯 产品级目标

任意新客户（运营人员的 / 客户员工的 / 外部公司的）通过这个命令一次性完成全部接入。

---

## 第 1 步：信息采集（自然语言交互）

依次问用户（**一次问一组，等回复再问下一组**，避免信息丢失）：

### 第 1 组：基本信息
- 客户公司全名
- 客户简称 / 内部 ID（如 `client-E`）
- 主要联系人 + 微信/WhatsApp
- 行业（自动匹配 [模式库/行业蓝图/](../模式库/行业蓝图/)，如 B2B 工业化工 / 跨境 B2C 时尚 / SaaS / 服务业）

### 第 2 组：网站信息
- 网站 URL
- 主要市场国家（影响 hreflang / GBP / 文化适配）
- 主要语言 + 扩展语言（如 EN / EN+FR/ES/PT / 6 语种）
- 平台（WordPress / Astro / 其他）
- 是否已上线（已上线 = 运营接管 / 在建 = 等建站交付 / 全新 = 切建站智能体）

### 第 3 组：现有数据访问
- GSC 是否已开通（service account email 加 Full User？）
- GA4 measurement ID
- IndexNow API key（如已部署）
- 是否有原 SEO 报告 / Audit 数据

### 第 4 组：业务目标
- 主要 KPI（询盘数 / 流量 / 排名 / 转化率？）
- 当前每月询盘数（baseline）
- 期望 6 个月达成（流量 / 询盘 / 排名）
- 客户独有的运营约束（不发某语种 / 不上某平台 / 等）

---

## 第 2 步：智能体自动执行（10 个动作并行）

**用户给完信息后立刻并行跑**：

### A. client-manager 建档
```
mcp__client-manager__add_client({
  name, id, industry, website,
  contact, languages, markets,
  status: "onboarding",
  goals, constraints
})
```

### B. 跑全站 SEO audit（**并行**）
```
seo-checker MCP:
- check_seo($URL)
- check_pagespeed($URL)
- check_hreflang($URL)（如多语言）
- check_structured_data($URL)
```

### C. 拉 GSC 基线（**并行**，需用户先授权）
```
search-analytics MCP:
- gsc_search_performance（30 天）
- gsc_index_status
- gsc_crawl_errors
- gsc_submit_sitemap（看 current_sitemaps，**重点查 WP 老 sitemap 残留**，参考[失败教训 #1](../模式库/失败教训/旧WP-sitemap-未清-展示量锁死.md)）
```

### D. 拉 GA4 基线
```
search-analytics MCP:
- ga4_traffic_summary
- ga4_traffic_sources
- ga4_conversions
```

### E. 应用对应行业蓝图
- 读 [模式库/行业蓝图/](../模式库/行业蓝图/) 对应文件
- 提取必装基础设施清单 + 90 天上手计划

### F. 应用所有相关失败教训
- 读 [模式库/失败教训/](../模式库/失败教训/) 全部
- 自动跑对应检查（如 WP→Astro 迁移客户必查老 sitemap）

### G. 应用相关成功模式
- 读 [模式库/成功模式/](../模式库/成功模式/) 全部
- 推荐适用于该客户的 Top 3

### H. 配 daily-cron 加新客户
- 修改 `mcp-servers/wecom-bot/daily-cron.mjs`，把新客户加进 active 列表
- pm2 不需重启（next 8:00 cron 自然加载）

### I. 部署客户站脚本（如客户站本地可访问）
- `scripts/index-boost.mjs`（IndexNow）
- `scripts/competitor-radar.mjs`
- `scripts/topic-pool.mjs`
- `scripts/ctr-opportunities.mjs`
- `scripts/geo-opportunities.mjs`

如脚本已存在则跳过，不存在从模板复制。

### J. 启动监控
- site-monitor MCP `check_site` 立即跑
- pm2-health-monitor 自动监控（已 cron）

---

## 错误处理

- 客户未给 GSC 授权 → 跳过 audit C，标记"等待客户授权"
- audit 发现客户站不可访问 → 立即报告（可能未上线 / DNS 问题）
- 行业不匹配现有蓝图 → 用"通用蓝图"+ 标记"需新建该行业蓝图"

---

## 注意

- 这个命令是**非破坏性**的（只读 + 建档，不动客户站本身）
- 所有自动化结果都要人工审核后才执行实际改动（如老 sitemap 删除）
- 客户欢迎包草稿需运营人员/项目经理审一遍再发
