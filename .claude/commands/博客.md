激活网站运营智能体 v10，给客户 **$1** 出一篇博客。

## 🎯 参数处理（v10 升级 — 数据驱动选题）

**情况 A：用户传了 $2（指定主题模式）**
- 主题 = $2，跳过自动选题，直接进入内容生产 12 阶段流程
- 仍需用 seo-sxo skill 反向读 SERP 验证主题 page-type 是否适合博客

**情况 B：未传 $2（自动选题模式 — 推荐这个，天花板姿态）**

执行**数据驱动选题 5 步**：

### 自动选题 5 步

1. **跑 topic-pool.mjs**（如客户站 `scripts/topic-pool.mjs` 存在 — demo-c + demo-b 已有）
2. **拉 GSC 内容缺口**（search-analytics MCP `gsc_search_performance` 拿"高展示低排名 query"，过滤带 -site: 假关键词）
3. **拉竞品最新动作**（`scripts/competitor-radar.mjs` 看竞品 7 天内新文章）
4. **拉 CTR 机会**（`scripts/ctr-opportunities.mjs` 看高排名低 CTR 页 — demo-c + demo-b 已有）
5. **综合评分排序**（按"潜力分 = 月搜索量 × 商业价值 × 竞品稀疏度"），出 Top 3 候选

### 候选输出格式（必须给运营人员过目）

```
🎯 自动选题 Top 3（基于 [日期] 数据）：

1. [主题 X]
   - 数据：月搜索量 ~XXX / 当前位置 ~Y / 竞品覆盖度 Z
   - 商业价值：⭐⭐⭐⭐ (能带询盘)
   - 推荐理由：[一句话]

2. [主题 Y] ...
3. [主题 Z] ...

我建议先写 Top 1。要不要立刻开始？
（说"开始" / "换 Top 2" / "我自己定主题: XXX"）
```

**等运营人员点头**（避免 AI 跑错主题写一半浪费 token）→ 进入内容生产 12 阶段

---

## 📝 内容生产 12 阶段（A 或 B 路径都要跑）

执行 `.claude/skills/content-production.md` 全流程：

1. **选题验证**：
   - search-analytics MCP 看该关键词 GSC 是否已有展示数据（避免重复内容）
   - seo-sxo skill 反向读 SERP 验证 page-type
   - 如果 SERP 是产品页/对比页主导 → 警告博客文章突破不了，建议改 page-type

2. **内容生产**（确认是文章型才继续）：
   - 标题（含主关键词，过 humanizer-zh 第 1/2/8 项规则）
   - 段首 60 字给定义（GEO 优化）
   - 1-2 个 200-260 字"自包含答案块"（seo-geo skill）
   - H2/H3 问句式
   - 具体统计数据 + 来源
   - FAQ 区块 3-5 问
   - 内链 3-5 个相关页面

3. **SEO 元数据**：title ≤ 60 字、desc 150-160 字、canonical、og:image、Schema Article + Person

4. **强制过 humanizer-zh** 去 AI 味（24 种模式检测）

5. **图片**：按"图片红黄绿三线规则"决定 → image-generator MCP `generate_cover` 出主图

6. **发布前 QA**：seo-checker MCP `check_seo` 自检；多语言站调 `check_hreflang` 8 项

7. **time line**：client-manager 记录"今日发布博客《主题》"

**输出**：完整博客 markdown + 主图 + 元数据 + 客户预览说明 + 7 天后效果回查日期
