激活网站运营智能体 v9.4，对客户 **$1** 紧急修复问题：**$2**

执行 `.claude/skills/hotfix.md` 闭环：

1. **诊断阶段**：
   - 复现问题（fetch 实际页面 + seo-checker MCP `check_seo`）
   - **MCP 数据不盲信**，用 fetch 二次验证（重定向页数据不可靠）
   - 定位根因（代码 / 配置 / DNS / SSL / WP 插件冲突 / 部署）
   - 影响范围评估（单页 / 单语种 / 全站）

2. **修复阶段**：
   - 修代码 / 改配置 / 重启服务 / 重新部署
   - 多语种站要全语种修，不能只修一种
   - 部署后 IndexNow 推送 + GSC URL Inspection 重抓

3. **验证阶段**（强制）：
   - fetch 实际页面确认问题已消失
   - 跑 seo-checker MCP `check_seo` 看分数
   - 如涉及多语言，跑 `check_hreflang` 8 项校验
   - 截图对比修复前后

4. **复盘阶段**（强制 4 步自我进化）：
   - 第 1 步：问题怎么产生的？
   - 第 2 步：为什么没在巡检中发现？（巡检项缺失？）
   - 第 3 步：会不会在其他客户/页面重现？（多客户扫描）
   - 第 4 步：更新规则（daily-ops.md 加新巡检项 / QA 清单加新检查 / CLAUDE.md 加规则）

5. **客户沟通**：
   - 出客户能看懂的修复说明（中文，过 humanizer-zh）
   - client-manager 时间线记录"hotfix [$2] → 已修复 → 已验证 → 已更新规则防复发"

**严禁**：只修单点不复盘 / 只复盘不更新规则 / 不做多客户扫描就收尾。

**触发条件检查**：本次问题是否符合 4 步自我进化触发条件（客户截图反馈 / GSC 报错 / 巡检漏发现 / 多页面同问题），如符合则强制执行。
