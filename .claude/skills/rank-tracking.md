# 关键词排名追踪技能

> 触发指令：`[客户名] 排名追踪` / `[客户名] 排名报告`  
> 执行频率：每周1次（配合周检）  
> 人工确认：无需确认，自动输出

---

## 数据来源

### 优先级1：search-analytics MCP（实时GSC API数据）
```
→ search-analytics.gsc_search_performance(site=域名, dimension='query')
→ search-analytics.gsc_ranking_changes(site=域名)
→ 直接从Google获取真实排名数据，无需CSV
```

### 优先级2（备用）：GSC CSV数据
```
客户-XX/数据/YYYY-MM_GSC_查询报告.csv
→ 仅在API不可用时使用
```

## MCP 调用链

```
Step 1: 加载客户
        → client-manager.get_client(client_id)
        → 获取域名、核心关键词列表

Step 2: 拉取实时排名数据（全自动）
        → search-analytics.gsc_search_performance(site=域名, days=7, dimension='query', limit=50)
        → search-analytics.gsc_search_performance(site=域名, days=28, dimension='query', limit=50)
        → 获取关键词的点击/展示/CTR/排名

Step 3: 排名变化分析（全自动）
        → search-analytics.gsc_ranking_changes(site=域名)
        → 自动对比本周vs上周：提升/下降/新增/消失
        → 排名变化>5位的关键词自动标红

Step 4: 核心关键词排名分析
        → 提取核心关键词（产品词+行业词+品牌词，通常15-30个）
        → 计算每个词：本期排名 | 上期排名 | 变化 | 展示量 | 点击量 | CTR

Step 4: 趋势分类

    📈 上升（排名提升>=3位）
    → 这些词在起效，继续强化对应页面

    ➡️ 稳定（变化<3位）
    → 维持现状

    📉 下降（排名下降>=3位）
    → 需要诊断原因并干预

    🆕 新增（上期无数据，本期出现）
    → 新内容开始获得排名

    ❌ 消失（上期有数据，本期无）
    → 紧急：页面可能被移除索引

Step 5: 机会关键词识别
        → 筛选：排名8-20位 + 展示量>50
        → 这些词"差一点就到首页"，是最值得投入优化的

Step 6: 竞品排名对比（如有竞品关键词数据）
        → fetch竞品页面，对比同一关键词的内容深度
        → 识别竞品排名上升的关键词

Step 7: 输出排名报告
        → 保存到 客户-XX/数据/YYYY-WXX_排名追踪报告.md
        → client-manager.add_timeline(id, "排名追踪: X个词上升/X个下降")
```

## GSC数据使用规范（必读）

> 来源：客户B实战教训——向客户报"排名第1"，客户搜不到，信任受损。

1. **过滤假关键词**：query 中包含 `-site:` 或引号+排除条件的，必须从报告中剔除
2. **排名用"平均排名"**：position 6.5 不代表稳定在第6名，报告中标注"GSC平均排名"
3. **禁止夸大表述**：禁用"登顶""排名第1""已进入首页"。用"GSC显示平均排名X.X"
4. **区分词类**：长尾词（4+单词）排名预期与核心商业词（1-2词）完全不同，不混谈
5. **客户可验证声明**：报告末尾加注"排名因搜索地区、语言、设备不同而异，以GSC数据为准"

## 核心关键词管理

### 关键词分类

| 类型 | 数量 | 示例 | 监控频率 |
|------|------|------|---------|
| 品牌词 | 3-5个 | Demo-C, demo-c.com | 每周 |
| 核心产品词 | 10-15个 | EPS block molding machine, EPS pre-expander | 每周 |
| 长尾内容词 | 10-20个 | how to choose EPS machine, EPS vs EPP | 每两周 |
| 竞品对比词 | 5-10个 | EPS machine manufacturer China | 每周 |

### 关键词基线建立

首次执行时：
1. 从GSC CSV提取所有有展示的查询词
2. 筛选展示>10的词作为追踪词
3. 记录初始排名作为基线
4. 保存到 `客户-XX/数据/关键词基线.md`

后续每次追踪与基线对比，显示从启动至今的总变化。

|------|------|
| 关键词排名下降 | → content-refresh 检查对应文章是否需要更新 |
| 机会关键词（11-20位） | → content-production 针对性写文章/优化现有页面 |
| 新关键词上榜 | → data-analysis 纳入下月选题参考 |
| 关键词消失 | → hotfix 紧急检查页面是否被移除索引 |
| 竞品同词排名上升 | → competitor-monitor 深度分析竞品策略 |

## 无GSC数据时的替代方案

当客户尚未提供GSC CSV时：

1. **基于内容推断核心关键词**
   → 读取所有博客标题和描述 → 提取主关键词
   
2. **基于MCP检测替代**
   → seo-checker.check_seo(关键页面) → 获取当前SEO状态
   → 与上次检测对比，判断趋势

3. **引导客户导出GSC数据**
   → 输出操作指南（截图级步骤）
   → 文件命名和放置路径说明

**关键提醒**：无GSC数据时，排名追踪的精度大幅下降。应优先推动客户完成GSC数据导出。
