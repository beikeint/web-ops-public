# content-distribution — 1 博客 → 50 社媒裂变流水线

## v12.0 战略产物

运营人员确认 2 档代运营产品 + 社媒纳入产品体系（2026-05-07）。本 skill 是产品交付核心 — **每篇深度博客 = 1 个内容核 → 自动派生 N 个二级内容渠道形态**。

**关键认知**：你不用多写一份原料，只是同一份内容**裂变形态**到不同渠道。

## 2 档差异

### 标准档 — 1 渠道选择
客户从下面 3 个选 1：
- **LinkedIn 公司页** — 海外 B2B 出海客户首选
- **微信公众号** — 国内品牌客户首选
- **小红书** — B2C 行业 / 设计 / 化工美妆类首选

每篇深度博客派生 5-10 个二级内容（按渠道特性）。

### 旗舰档 — 6 渠道全覆盖
- **LinkedIn**（个人 IP + 公司页）
- **YouTube**（视频频道）
- **Twitter/X**（行业讨论）
- **微信公众号**（中文版）
- **小红书**（场景化短帖）
- **知乎**（行业问题真账回答）

每篇深度博客派生 30-50 个二级内容 + 客户老板个人 LinkedIn 同步。

## 派生流水线（18 阶段深度博客后自动跑）

### 阶段 1：内容核拆解
从 5000 字博客中提取：
- **TL;DR**（150 字摘要）
- **3-5 个核心数据点**（每个 50 字 + 数字）
- **3-5 个 H2 子论点**（每个 200 字独立可读）
- **5 个 FAQ**（每个 100-150 字）
- **真客户案例**（300-500 字独立可读）
- **专家 quote**（30-50 字）
- **配图素材**（3-6 张原创图）
- **视频素材**（30 秒主视频 + 多个片段）

### 阶段 2：按渠道形态派生

#### LinkedIn（公司页 / 个人 IP）

| 形态 | 来源 | 频次 |
|---|---|---|
| 长帖 (1500 字) | 博客 TL;DR + Top 3 论点 | 月度 1-2 |
| 短帖 (250 字) | 单个数据点 / FAQ | 周度 2-3 |
| Carousel (10 卡) | H2 子论点拆 | 月度 1 |
| Video (30s-2min) | 视频素材 | 月度 1 |
| 文档 PDF | 博客 PDF 版 | 月度 1 |
| **派生总数** | | **8-12 帖/月** |

#### YouTube

| 形态 | 来源 | 频次 |
|---|---|---|
| 主视频 (3-5min) | 完整工厂实拍 + 数据图表 | 月度 1 |
| Shorts (15-60s) | 视频片段切割 | 月度 4-6 |
| 播放列表 | 行业系列 | 季度规划 |
| **派生总数** | | **5-7 视频/月** |

#### Twitter/X

| 形态 | 来源 | 频次 |
|---|---|---|
| 线程 (10 推) | H2 子论点 + 数据 | 月度 1-2 |
| 单推 | FAQ / 数据 | 周度 3-5 |
| 行业讨论引战 | 客户案例 / 专家观点 | 月度 1 |
| **派生总数** | | **15-20 推/月** |

#### 微信公众号

| 形态 | 来源 | 频次 |
|---|---|---|
| 中文深度文 (2000 字) | 博客中文翻译 + 本地化 | 月度 1 |
| 短文 / 链接卡 | 案例 / 数据 | 周度 1 |
| **派生总数** | | **5-6 文/月** |

#### 小红书

| 形态 | 来源 | 频次 |
|---|---|---|
| 场景化短帖 (500 字 + 9 图) | 客户案例 + 配图 | 周度 1-2 |
| 工厂实拍系列 | 视觉素材 | 周度 1 |
| **派生总数** | | **8-12 帖/月** |

#### 知乎

| 形态 | 来源 | 频次 |
|---|---|---|
| 行业问题真账回答 (1500 字) | 博客 H2 + FAQ | 周度 1-2 |
| 短问答 | 单个 FAQ | 周度 2-3 |
| **派生总数** | | **10-12 答/月** |

### 阶段 3：渠道适配 + 本地化

每个二级内容针对渠道特性调整：
- LinkedIn：B2B 专业语气 + emoji ≤ 3 + 标签 ≤ 5
- YouTube：标题 ≤ 60 字符 + 描述含 5 个相关 hashtag
- Twitter：简短 + 行业专家 @
- 微信公众号：中文化 + 本地案例替换
- 小红书：场景化 + emoji ≤ 8 + 9 图轮播
- 知乎：问题导向 + 真账数据

### 阶段 4：发布排期

| 渠道 | 节奏 |
|---|---|
| LinkedIn 公司页 | 每周二 / 周四 |
| LinkedIn 个人 IP（旗舰档）| 每周一 / 周三 / 周五 |
| YouTube | 每月 5 号主视频 / 10、15、20、25 号 Shorts |
| Twitter | 每周一-五 各 1 条 |
| 微信公众号 | 每周三 主文 |
| 小红书 | 每周二、五 |
| 知乎 | 每周一、四 |

### 阶段 5：跨渠道反向引流

- LinkedIn / Twitter 帖 末尾 → 引流网站博客（用 UTM 跟踪）
- YouTube 视频 → 描述区 → 网站对应博客深度版
- 微信公众号 → 文末扫码 → 网站联系页
- 小红书 / 知乎 → 主页签名链接 → 网站

### 阶段 6：业务结果跟踪

每渠道月度 KPI：
- LinkedIn：粉丝 / 互动率 / 点击网站 UTM 流量
- YouTube：订阅 / 播放 / 完播率
- Twitter：粉丝 / 转推 / 点击
- 微信公众号：阅读 / 在看 / 点击文末链接
- 小红书：粉丝 / 收藏 / 点击主页签名
- 知乎：赞同 / 收藏 / 关注

## MCP 调用链

```
Step 1: 阶段 1 内容核拆解（提取 TL;DR / 数据 / FAQ / 案例 / quote）
Step 2: 阶段 2 按选定渠道派生（标准档 1 / 旗舰档 6）
Step 3: 渠道 MCP 发布
        - LinkedIn: mcp__linkedin__linkedin_create_post + 多 image / video / poll 变体
        - YouTube: 暂无 MCP，输出脚本到 docs/social-distribution/youtube-{date}/
        - Twitter: 暂无 MCP，输出脚本
        - 微信公众号: mcp__wechat-publisher__wechat_publish_draft
        - 小红书: 暂无 MCP，输出脚本
        - 知乎: 暂无 MCP，输出脚本
Step 4: 排期 cron - new monthly-social-distribution-cron
Step 5: 跨渠道 UTM 跟踪 - GA4 ga4_traffic_sources
Step 6: 月度 ROI 跟踪 - business-outcome-tracker 集成
```

## 容量约束（一人公司硬约束）

- 标准档 1 渠道 = 全自动化跑，可服务 30+ 客户
- 旗舰档 6 渠道 = 需运营人员本人协调（采访 / quote 提取 / 客户老板沟通），3-5 客户上限

## 反元规则

- 不要为"完成派生数"凑数（每个二级内容必须独立可读，不允许"我们刚发了博客 X，请看 [link]"这种水帖）
- 不要在 LinkedIn / Twitter 等渠道过度自吹（"行业领先"等词触发 SpamBrain 类似算法）
- 多渠道内容**完全相同 100% 复用**会被算法降权，必须每渠道独立改写形态
- 不要替客户老板"假装是他写的" — 旗舰档 LinkedIn 个人 IP 帖必须客户老板审过才能发
