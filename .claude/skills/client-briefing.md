# 客户日报简报

> 触发指令：在 daily-ops + daily-growth 执行完毕后自动生成 / `[客户名] 生成今日简报`  
> 执行时机：每日工作流的最后一步  
> 人工确认：无需确认，直接输出可发送文本

---

## 为什么需要客户日报

1. **客户看得到你每天在工作** — 续费率直接拉满
2. **建立专业信任** — 每天一条简报比月底一份厚报告更有冲击力
3. **降低沟通成本** — 客户不用问"最近做了什么"

---

## 简报生成规则

### 格式要求

- **中文**，适合发微信群
- **3-5行**，手机一屏看完
- **不要技术术语**，客户是老板不是SEO专家
- **有数字**，让成果可感知
- **有表情符号**，微信群里显得更亲切自然
- 末尾固定签名：`—— 独立站建站智能体·AI网站运营`

### 内容结构

```
[日期 + 网站状态一句话]
[今日巡检：做了什么/发现什么/修了什么]
[今日增长：做了什么具体优化]
[一句话预告明天方向]
签名
```

## MCP 调用链

```
Step 1: 汇总今日工作
        → 从 daily-ops 的执行结果中提取：
          - 网站状态（正常/异常）
          - 发现的问题
          - 修复的问题
        → 从 daily-growth 的执行结果中提取：
          - 做了什么优化
          - 针对哪个页面
          - 预期效果

Step 2: 选择模板
        → 根据今日工作内容选择最匹配的模板
        → 填充具体数据

Step 3: 语言优化
        → 技术术语转为客户能理解的表达：
          "优化Title标签" → "优化了搜索标题"
          "补充FAQ Schema" → "新增了常见问题版块"
          "内链建设" → "建立了文章之间的关联链接"
          "CTA优化" → "优化了询盘按钮"
          "Core Web Vitals" → "页面加载速度"
          "排名11→8" → "关键词排名提升了3位"
          "GEO优化" → "提升AI搜索收录率"
          "Featured Snippet" → "搜索结果精选展示位"

Step 4: 输出
        → 直接输出可复制发送的文本
        → 不保存文件（简报是即时消息，不是档案）
```

### GSC数据使用规范（必读）

> 来源：客户B实战教训——向客户报"排名第1"，客户搜不到，信任受损。

1. **过滤假关键词**：query 中包含 `-site:` 或引号+排除条件的，必须从报告中剔除
2. **排名用"平均排名"**：position 6.5 不代表稳定在第6名，报告中标注"GSC平均排名"
3. **禁止夸大表述**：禁用"登顶""排名第1""已进入首页"。用"GSC显示平均排名X.X"
4. **区分词类**：长尾词（4+单词）排名预期与核心商业词（1-2词）完全不同，不混谈
5. **客户可验证声明**：报告末尾加注"排名因搜索地区、语言、设备不同而异，以GSC数据为准"

---

## 注意事项

1. **不说负面的事**：如果今天修了bug，说"发现并修复了X个问题"而不是"网站出了X个问题"
2. **强调主动性**：用"我们今天优化了""我们发现并立即处理了"而不是被动语态
3. **有未来感**：每条简报最后一句预告明天做什么，让客户觉得工作是持续推进的
4. **不过度承诺**：说"预计提升"而不是"一定会提升"
5. **简短**：5行以内，微信一屏看完。客户很忙，不要长篇大论

## 客户常见问题话术库

```

下一阶段（排名稳定后）：需要您配合提供4样东西——
① 3-5个成功案例的照片+一句话描述
② 客户评价或反馈截图
③ 工厂/发货实拍照片
④ 买家常问的问题

到时候我会提前跟您说，现在您什么都不用准备，我们先把基础打好 👍
```
