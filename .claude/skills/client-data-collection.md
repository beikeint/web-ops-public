# client-data-collection — Path C 90+ 内容生产的硬阻塞

## v11.0+1 (2026-05-07 立) Path C 战略基础

运营人员确认 Path C：「目标导向，可以要求客户必须配合，不能因为需要客户配合就放弃这条路」。

网站运营智能体产品定位升级：
- ❌ 旧定位：「我们帮你做 SEO」
- ✅ 新定位：**「我们帮你做行业前 1% 的内容质量，但需要你深度参与」**

**这不是恳求客户配合，是产品强制要求**。客户不配合 = 不接 / 不交付 90+。

## 4 类硬资源（每月 / 客户必给）

### 资源 1：真运营数据点（≥ 3 个/月）

**形式**：客户从生产 / 财务 / 销售 / 运营系统拿到的真数字。
**例**（按行业）：
- Demo-D：「2025 年我们生产 380 万对耳塞」/「单条产线月产能 12 万只」/「主力产品 NRR 33dB（实测 5000 次抽样）」
- EPS：「BM-1400 单台年产能 8000m³ EPS 块材」/「90 天回本周期实测 12 个客户」
- 冷链：「单台 PU 喷涂机日产 280 个保温箱」/「平均 R 值 5.3 实测」
- Demo-B：「PVA 胶月产能 2000 吨」/「EN 204 D3 通过率 99.7%」/「主力客户 ¥ 800 万年订单」

**用途**：嵌入博客每 300 字 ≥ 1 个具体数字。

### 资源 2：真实客户案例（≥ 1 个/月）

**形式**：客户给的真实下游买家成功故事。
**必含**：
- 具体公司名（可匿名为"中东某 EPS 厂"但要有国家+规模）
- 具体行业 / 应用场景
- 具体收益（产能 +X% / 成本 -¥Y / 投资回收 N 月）
- 客户的一句反馈（30-50 字）
- 时间（月份）

**用途**：博客中至少 1 节叫"Real-world case"或类似，引用真案例。

### 资源 3：内部专家 LinkedIn quote（≥ 1 个/月）

**形式**：客户员工真名 + LinkedIn 个人页 + 30-50 字技术 quote + 头像照（可选）。
**例**：「张工，Demo-D R&D 总监，linkedin.com/in/zhang-eng/，10 年 PPE 设计经验」+ 一句 quote 如「Foam ear plug NRR 测试时，环境温度的影响经常被低估，建议买家追问 NIOSH ANSI 测试条件」。

**用途**：
- 博客 author / authorUrl 字段填该专家
- 博客中嵌入该专家的 quote 段（带 LinkedIn 链接）
- Person Schema sameAs 用 LinkedIn URL

### 资源 4：原创视觉资产（≥ 3 张/月）

**形式**：工厂实拍 / 产品对比图 / 流程图 / 数据图表。
**禁止**：网络下载图 / 库存图 / AI 生成无水印图。
**用途**：
- 博客内文配图（≥ 3 张）
- 主图（cover）
- 信息图（infographic）

## 收集流程

### 月初（每月 1-3 号）

发"月度内容深度合作通知"给客户老板（[案例库/Path-C-客户通知模板.md](../../案例库/) v11.0+1 同步）：

```
{{客户老板名}} 您好，

{{月份}} 我们计划为贵公司站发 1 篇行业前 1% 的深度博客 + 4-6 篇基础博客。
深度博客需要您配合提供 4 类资源（5 月 1-7 号截止）：

1. 1-3 个真运营数据点（产能 / 收益 / 实测数据）
2. 1 个真实客户案例（脱敏可，含国家+具体收益）
3. 1 个内部专家配合（员工真名 + LinkedIn）
4. 3 张原创视觉（工厂实拍 / 产品对比 / 流程图）

资源齐全后 7-10 天内交付深度博客（投资回报：通常带来 5-15 个高质量询盘 / 月，
按行业差异）。资源不到位则跳过本月深度博客，仅交付基础内容。

不便提供请回复"本月跳过"。

独立站建站智能体
```

### 客户回复后

- **资源齐全** → 启动 [deep-content-production](deep-content-production.md) 12+6 阶段
- **资源部分到位** → 重新协商：能否延期 / 能否补部分（不允许"凑数"开始）
- **客户拒绝配合** → 本月跳过深度博客 + client-manager.add_timeline 记 'Path C 跳过 - 客户拒配合 N 个月'
- **连续 3 月跳过** → 升级议题（运营人员决策是否调整服务模式 / 价格 / 终止）

### 资源归档

每客户每月一份资源包：

```
客户/<客户>/website/docs/path-c-resources/
├── YYYY-MM/
│   ├── 资源-1-运营数据.md
│   ├── 资源-2-客户案例.md
│   ├── 资源-3-内部专家.md（含 LinkedIn / quote / 头像路径）
│   └── 资源-4-视觉/
│       ├── factory-1.jpg
│       ├── product-compare.jpg
│       └── process-flow.svg
```

资源包是深度博客生产的**唯一启动凭证**，没资源包不启动 deep-content-production。

## 硬规则（不容协商）

1. **资源不到不启动**：deep-content-production 阶段 0.5 必读资源包，缺任一类直接 abort
2. **客户拒配合不施压**：发通知后不催不缠，第一次拒 = 本月跳过 / 三次拒 = 升级议题给运营人员
3. **资源真实性 ≥ 数量**：客户给"通用数据"或"模板案例"= 视为没给（拒收，告诉客户"我们要的是真账，不是文案模板"）
4. **保密保护**：资源包仅本智能体使用 + 不外发 + 不进 release-agent.sh 推送（同 .env 级保护）
5. **跨客户复用禁止**：A 客户案例不能给 B 客户用（保证真实性）

## 客户拒配合时运营人员的决策矩阵

| 客户态度 | 我的处置 | 运营人员决策 |
|---|---|---|
| 第 1 次拒 | 本月跳深度博客，继续基础内容 | 无需介入 |
| 第 2 次拒 | 同上 | 无需介入 |
| 第 3 次拒 | client-manager.add_timeline P0 警示 | **必须介入**：要么客户被说服 / 要么调整服务定位（基础包 vs 深度包） |
| 客户给假数据 | 一次警告 + 第二次终止 | 必须介入 |

## 配套：客户合同 / 服务协议条款（运营人员法务侧）

新接入客户必须签 Path C 配合条款（运营人员客户经理智能体协助起草）：

```
《内容深度合作配合条款》

为达成行业前 1% 内容质量目标，乙方（客户）承诺每月配合甲方（网站运营智能体）提供：
1. ≥ 3 个真实运营数据点
2. ≥ 1 个客户案例
3. ≥ 1 个内部专家 LinkedIn quote
4. ≥ 3 张原创视觉资产

连续 3 月不配合则甲方有权降级至基础服务（仅 75-85 分博客 / 月度报告）。
```

## 与现有机制的关系

- **替代**：client-onboarding 加"Path C 配合协议"步骤（同步更新）

---

## 反元规则（防止本 skill 漂移）

- **不要变成"恳求客户"**：本 skill 是产品强制要求，不是低声下气
- **不要"客户给啥用啥"**：质量底线在我们这边，客户给假/凑数的资源直接拒收
- **不要为"完成 KPI"凑数**：每月 1 篇深度博客是目标不是死指标，客户不配合宁可跳月也不降标准
