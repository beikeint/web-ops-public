# 每日主动增长引擎

> 触发指令：`[客户名] 今日任务` / 每日巡检后自动衔接  
> 执行频率：每天1次，完成1-3个微任务  
> 人工确认：无需确认，自动执行+部署+验证+记录  
> 核心理念：**每天进步1%，一个月30个微优化，竞品不可能追上这个密度**

---

## 一、任务选择引擎（数据驱动，不拍脑袋）

每次执行时，按以下优先级自动选择今日任务：

```
Priority 0: 周检/巡检发现的未修复问题 → 先清债
Priority 1: 排名11-20位的关键词对应页面 → 差一步到首页，ROI最高
Priority 2: 高展示低CTR的页面 → 有曝光但没人点，Title/Desc有问题
Priority 3: 高流量高跳出的页面 → 来了人但留不住，内容或体验问题
Priority 4: 无FAQ/无Schema的高价值页面 → GEO缺口，补了就能被AI引用
Priority 5: 竞品本周新动作对应的主题 → 48小时内响应
Priority 6: 内容评分最低的页面 → 拖后腿的短板
Priority 7: 按星期轮换兜底 → 上面都没有时的默认任务
```

### MCP 数据采集（选择依据）

```
→ 读取最近一次 weekly-check 报告 → 未修复项
→ 读取最近一次 rank-tracking 报告 → 机会关键词（11-20位）
→ 读取 GA4 数据（如有）→ 高跳出/低停留页面
→ content-tracker.list_content → 评分最低的文章
→ 读取最近一次 competitor-monitor → 竞品新动作
→ 如以上数据都没有 → 按星期轮换
```

## 二、7大增长维度

### 维度1：SEO精细化（周一）· CTR 优化引擎驱动

**2026-04-20 升级**：从"手感改 Title"→ **数据驱动的 CTR 优化引擎**（见 `ctr-optimization.md` 技能）。

**默认执行流程**（每个活跃客户跑一次）：

```
Step 1: 跑机会页扫描（每个客户站 scripts/ctr-opportunities.mjs）
        → 输出 Top 5 机会页到 docs/ctr-opportunities-YYYY-MM-DD.md
        → 按"潜力分数 = 展示 × (期望CTR - 当前CTR)"排序

Step 2: 选 Top 1 做手术
        → 读当前 Title/Desc
        → 拉该页 top 3 真实 query（过滤 -site: 假查询）
        → 按 3 种策略（数字锚点 / 对比 / 长尾问题）生成候选
        → 按"含关键词+含数字+含 CTA+长度 50-65字符"评分选最优

Step 3: 部署 + 记录
        → 改 names.en / descriptions.en / Title 模板（按情况）
        → 部署 + 线上验证 Title 生效
        → IndexNow + GSC sitemap 重提交
        → 写 docs/ctr-log.md 改动日志（改前/改后 Title + 待复盘日期）

Step 4: 周二复盘
        → 跑上周改动的 7 天后 CTR 对比（脚本自动找 ctr-log.md 到期条目）
        → 提升 > 1pt → 策略纳入模式库；无效 → 降权
```

**补充动作**（机会页扫描没覆盖的场景）：

| 任务 | 做什么 | 选择依据 | 预期效果 |
|------|--------|---------|---------|
| H1-H2关键词对齐 | 检查H1是否含主关键词、H2是否覆盖支撑关键词 | 排名11-20位的页面 | 排名提升 |
| URL规范化 | 过长slug缩短、移除无意义词（the/and/of） | 抽查 | 爬虫友好 |
| Canonical审计 | 多语言页面canonical是否正确指向自身 | 批量检查 | 避免重复内容惩罚 |
| 搜索意图校准 | 该页面的目标关键词搜索意图是信息/商业/交易？页面内容是否匹配？ | 排名下降的页面 | 排名恢复 |

**天花板操作**：模板级 Title 后缀压缩。如果所有产品页/博客都套用"{name} | 长品牌尾巴"，改一次模板 = 全站几十页 SERP 空间释放（2026-04-20 demo-c 全站 EN 产品页后缀从 38 字符压缩到 12 字符）。

### 维度2：内容深度进化（周二）

不是简单"补字数"，而是**让内容成为该主题的最权威来源**。

| 任务 | 做什么 | 选择依据 | 预期效果 |
|------|--------|---------|---------|
| 补充答案胶囊 | 首段40-60词直接回答核心问题（BLUF原则） | 无答案胶囊的博客 | 被AI引用概率+50% |
| 添加数据表格 | 把文字对比改为表格呈现 | 有对比内容但无表格的文章 | SERP特征抢占 |
| 补充独家数据 | 从客户真实业务中提取脱敏数据，竞品无法复制 | 纯理论无数据的文章 | E-E-A-T加分 |
| 内容蚕食检测 | 查是否有2篇文章争同一个关键词 → 合并或区分 | GSC中同一关键词对应多个URL | 集中权重 |
| 段落重构 | >120词段落拆分，长句改短句 | GEO评分低的文章 | AI可读性提升 |
| 过时内容更新 | 文中"2024年数据""去年"等表述更新为最新 | 发布>90天的文章 | 内容新鲜度 |

**天花板操作**：内容蚕食检测。如果你有3篇文章都在争"EPS machine"这个词，Google不知道该给哪个排名，结果3篇都排不好。检测后要么合并为1篇超级文章，要么给每篇指定不同的子关键词。

### 维度3：内链架构优化（周三）

不是随机加链接，而是**构建让Google理解你主题权威性的链接网络**。

| 任务 | 做什么 | 选择依据 | 预期效果 |
|------|--------|---------|---------|
| 孤岛页面救援 | 找到0入链的页面，从3篇相关文章添加指向它的内链 | 内链矩阵中入链=0的页面 | 页面被发现和排名 |
| 支柱页强化 | 确保所有子文章都链回支柱页 | topic-cluster路线图 | 支柱页权威性提升 |
| 锚文本优化 | "click here"改为含关键词的描述性锚文本 | 全站扫描 | 关键词相关性信号 |
| 产品页→博客反向链接 | 产品页"了解更多"区域链接到相关博客 | 无出链的产品页 | 用户路径延长 |
| 断链修复 | 检查内链是否指向404或重定向 | fetch抽查 | 爬虫预算不浪费 |
| 爬虫路径优化 | 确保重要页面在首页3次点击内可达 | 检查导航结构 | Google更快发现新页面 |

**天花板操作**：爬虫路径优化。Google有"爬取预算"——它不会无限抓你的页面。确保高价值页面（产品页、核心博客）在首页2-3次点击内可达，而不是藏在5层子目录里。

### 维度4：GEO全面强化（周四）

不只是加FAQ，而是**系统性让每个页面都对AI搜索友好**。

| 任务 | 做什么 | 选择依据 | 预期效果 |
|------|--------|---------|---------|
| FAQ扩充+Schema | 每个高价值页面至少5个FAQ | 无FAQ的产品页/博客 | AI直接引用FAQ |
| llms.txt同步 | 新博客、新产品、新页面加入llms.txt | 发布新内容后 | AI爬虫快速发现 |
| 引用来源标注 | "据Grand View Research 2024报告"式标注 | 有数据但无来源的段落 | 可信度+引用概率 |
| 结论前置审计 | 每个H2首句是否直接回答本节问题 | 逐篇检查 | AI摘要匹配度 |
| 定义性语句植入 | "EPS pre-expander is a machine that..."式定义句 | 核心产品/术语页面 | 被AI引用为定义 |
| 对比矩阵标准化 | 所有对比内容用HTML表格呈现，而非纯文字 | 有"vs"/"对比"的文章 | AI提取结构化数据 |

**天花板操作**：定义性语句植入。当有人问AI"what is an EPS pre-expander"，AI需要找一个权威定义来引用。如果你的页面第一句就是精准的定义句，被引用的概率极高。在每个核心产品/术语页面的开头植入标准定义。

### 维度5：转化率优化/CRO（周五）

不是只优化CTA按钮颜色，而是**优化整个用户从进入到询盘的转化路径**。

| 任务 | 做什么 | 选择依据 | 预期效果 |
|------|--------|---------|---------|
| CTA文案优化 | "Contact Us"→"Get Your Free Factory Layout Plan" | GA4 cta_click率最低的页面 | 点击率提升 |
| 社会证明补充 | 添加"Exported to 30+ countries""CE/ISO certified" | 无信任信号的页面 | 转化率提升 |
| 退出挽留优化 | 检查高退出率页面，在退出前显示相关内容推荐 | GA4退出率最高的页面 | 跳出率降低 |
| 表单精简 | 减少非必要字段（如phone改为optional） | 联系页面 | 填写完成率提升 |
| 内容中间CTA | 在长博客的中间位置插入内联CTA | 阅读深度>50%的长文 | 抓住已深度阅读的用户 |
| 产品页询盘路径 | 产品页→询盘的路径是否清晰（不超过2次点击） | 高流量产品页 | 询盘转化率 |
| Thank-you页优化 | 询盘后推荐相关内容/提供下载资料 | thank-you页面 | 二次互动率 |

**天花板操作**：退出挽留+内容中间CTA。大多数访客看完一部分就离开了。在高退出点之前插入相关推荐或价值CTA，能把即将流失的流量变成询盘。

### 维度6：SERP特征抢占（周六上午）

不只是排名，而是**抢占Google搜索结果中的特殊展示位**。

| 任务 | 做什么 | 选择依据 | 预期效果 |
|------|--------|---------|---------|
| Featured Snippet抢占 | 找到Google已给出精选摘要的关键词，优化我们的页面格式匹配 | 排名前10但不是Snippet的词 | 抢占位置0 |
| People Also Ask渗透 | 从Google PAA中提取问题，加到我们博客的FAQ中 | fetch Google搜索结果提取PAA | 出现在PAA框中 |
| 表格Snippet优化 | 对比类内容用HTML表格，Google会直接显示在SERP中 | 有对比内容的页面 | SERP展示面积增大 |
| 列表Snippet优化 | 步骤类内容用有序列表，Google会显示为列表摘要 | 有"how to""步骤"的文章 | 抢占列表Snippet |
| 图片SEO强化 | 图片alt含关键词+图片文件名含关键词 | 产品图片 | 出现在Google图片搜索 |

**天花板操作**：Featured Snippet抢占。当你搜某个关键词，Google在第一条结果上方显示的那个框就是Featured Snippet。获得它的方法：用简短的段落（40-60词）直接回答该问题，放在相关H2标题下方。这就是答案胶囊的SEO价值。

|------|--------|---------|---------|

| 竞品外链突增 | 竞品获得高权威外链 | 外链监控（如有） | 争取同等来源外链 |

## 三、执行流程

```
Step 1: 数据采集（自动）
        → 读取最新周检/排名/竞品/GA4数据
        → 按Priority 0-7选择今日最高价值任务

Step 2: 任务规划
        → 输出：今日要做什么 + 为什么选这个 + 预期效果
        → 如有多个候选任务，选ROI最高的1-2个

Step 3: 执行
        → 读取目标文件 → 修改 → 构建验证

Step 4: 部署+验证
        → 部署到线上 → fetch验证修改生效

Step 5: 效果记录
        → client-manager.add_timeline
        → content-tracker.update_content（如修改了内容）
        → 记录修改前后对比，下周rank-tracking时回验效果

Step 6: 输出
        → 简洁报告：做了什么 + 为什么 + 修改前后 + 预期效果
```

## 五、30天增长效果模型

| 周 | 任务 | 累计影响 |
|----|------|---------|
| W1 | 5个SEO微调+5个内容增强 | 5个页面Title优化→CTR预期+2% |
| W2 | 5个内链+5个GEO强化 | 3个孤岛页面救援+8个FAQ补充 |
