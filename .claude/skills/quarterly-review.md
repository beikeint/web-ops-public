# 季度深度复盘技能

> 触发指令：`[客户名] 季度复盘` / `所有客户 季度复盘`  
> 执行频率：每季度末（3月/6月/9月/12月）

---

## MCP 调用链

```
Step 1: 加载客户信息
        → client-manager.get_client(client_id)
        → 获取所有timeline记录

Step 2: 数据汇总（并行）
        ┌→ content-tracker.list_content(client_id) → 本季度内容产出
        ├→ 读取本季度3份月报
        ├→ 读取本季度排名追踪数据
        ├→ 读取本季度AI引用监控数据
        └→ 读取GA4/GSC季度数据（CSV或API）

Step 3: 计算核心指标

    3a. 流量指标
        → 季度总会话数 vs 上季度
        → 自然搜索流量占比变化
        → 新用户 vs 回访用户

    3b. 排名指标
        → 首页关键词数量变化
        → 平均排名变化
        → 机会关键词转化率（从11-20位进入前10）

    3c. 内容指标
        → 本季度发布文章数
        → 平均内容评分
        → 内容集群完成度
        → 文章平均阅读深度

    3d. 转化指标
        → 询盘总数 vs 上季度
        → 询盘来源页面TOP5
        → CTA点击率变化
        → 询盘→成交转化率（如客户提供数据）

    3e. 技术指标
        → PageSpeed分数变化
        → 索引页面数变化
        → Schema覆盖率
        → 404修复数量

    3f. GEO指标
        → AI引用次数变化
        → llms.txt状态
        → FAQ覆盖率

    3g. 外链指标
        → 新增引用域名数
        → 外链质量评估

Step 4: ROI 计算
        → 本季度服务费用
        → 本季度可量化产出（流量增长、排名提升、询盘增长）
        → 如有成交数据：询盘→客户→合同额
        → 投入产出比

Step 5: 策略评估
        → 哪些策略有效（继续）
        → 哪些策略无效（调整/放弃）
        → 哪些新机会出现（纳入下季度）

Step 6: 下季度规划
        → 内容集群路线图更新
        → 技术优化重点
        → 外链建设目标
        → 关键词排名目标

Step 7: 输出 + 记录
        → 保存到 客户-XX/报告/YYYY-QX_[客户名]_季度深度复盘.md
        → client-manager.add_timeline(id, "Q[X]季度复盘完成")
```

### GSC数据使用规范（必读）

> 来源：客户B实战教训——向客户报"排名第1"，客户搜不到，信任受损。

1. **过滤假关键词**：query 中包含 `-site:` 或引号+排除条件的，必须从报告中剔除
2. **排名用"平均排名"**：position 6.5 不代表稳定在第6名，报告中标注"GSC平均排名"
3. **禁止夸大表述**：禁用"登顶""排名第1""已进入首页"。用"GSC显示平均排名X.X"
4. **区分词类**：长尾词（4+单词）排名预期与核心商业词（1-2词）完全不同，不混谈
5. **客户可验证声明**：报告末尾加注"排名因搜索地区、语言、设备不同而异，以GSC数据为准"
